# HTML

## 1. 앵커(anchor) 태그

- 표기 방법 : `<a>내용</a>`
- 서로 다른 웹 페이지 사이를 이동하거나 웹 페이지 내부에서 특정한 위치로 이동할 때 사용하는 태그다.
- 사용 방법
  - **내부링크** : 태그에 `id` 속성을 부여하여 사이트 내부로 이동
  >`id` 
    - 문서내에 요소(element)에 유일한 값을 부여한다.
    - 네이밍
      - 한글x 영어o
      - 띄어쓰기 x<BR>(JS코드가 들어가면 문제가 생긴다, 띄어쓰기를 쓰고싶으면 대신 hyphen을 쓴다.)
      - 언더바 x<BR>(주소창, 링크 밑 줄의 가시성 때문이다)
  - **외부링크** 
    - `target="_blank"` : 새 탭에서 외부링크가 열리게 하는 속성

<BR><BR>

## 2. 이미지 태그

- 표기 방법 : `<img src="소스">`
- 이미지를 html 문서에 넣는 태그다.
- 3가지 사용 방법
  - **내부링크** : 로컬내의 이미지를 불러오는 형식이다.
  - **외부링크** : 이미지의 주소를 불러오는 형식이다.
  - **데이터 url**
  >base64 방식으로 인코딩한 데이터를 소스 그대로 올리는 방법이다. <BR>과거 HTTP/1 시절 프로토콜의 제한으로 인해 http 요청을 줄이기 위한 해결 방법이었지만, 2021년 현재 HTTP/2 를 사용하는 환경에서는 해결이 되어 의미가 많이 퇴색된 기술이다. (심지어 현재 유튜브, 구글같은 곳에서는 http/3를 시범적으로 사용하고 있다.)
  - [2011년도 당시의 글](https://www.davidbcalhoun.com/2011/when-to-base64-encode-images-and-when-not-to/#commento-login-box-container){:target="_blank"}
  - 이미지가 저장된 서버가 훼손된다면 문제가 생기는 보수적인 상황에서는 html 문서 자체에 저장하는 이 방식이 도움이 될 것 같다. @확인필요

### 이미지 태그의 속성
- src : 이미지의 경로 지정
- width : 이미지의 너비 지정
  - pc는 1200 이하 태블릿과 작은 화면에서는 768~996 사이로 맞춰줌
- height : 이미지의 높이 지정
  - 높이와 너비의 비율을 정확히 알지 못할때는 한 가지 값만 조정하면 자동으로 비율이 맞춰진다.
- alt (alternative text)
  - 주소에 실제 이미지가 없을 때 출력
  - 이미지를 볼 수 없는 브라우저일 때 출력
  - 시각장애인 리더기에서 읽어주는 텍스트로 활용
  - 검색엔진에서 이미지를 수집할 때 사용하는 텍스트<BR>※ SEO는 sementic, alt 값을 잘 써야 한다.

### tip 파일 경로
  - `.` : 현재 디렉토리
  - `..` : 상위 디렉토리
  - `/` : 디렉토리간 구분
