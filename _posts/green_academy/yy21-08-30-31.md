flux
- FLux 클라이언트-사이드 관리 아키텍처
- Flux는 Facebook에서 클라이언트-사이드 웹 어플리케이션을 만들기 위해 사용하는 어플리케이션 아키텍쳐다. 단방향 데이터 흐름을 활용해 뷰 컴포넌트를 구성하는 React를 보완하는 역할을 한다. 
- https://haruair.github.io/flux/docs/overview.html#content
- https://facebook.github.io/flux/

Redux
- Flux 패턴을 라이브리러(JS/TS)로 만들어 놓은 것
- ActionCreater: 액션객체를 생성하는 함수
- Action: 액션타입(명령), 데이터(매개변수)
- Dispatcher: 액션타입과 페이로드를 받아서 리듀서를 실행
- Reducer: 기존State와 액션의 페이로드로 state를 변경

- https://ko.redux.js.org/introduction/getting-started/
- 커맨드 패턴 이라는 디자인 패턴으로 만들어져있다.

## `React-Redux`: React에서 Redux를 쓸 수 있게 만들어 놓은것
- 남발하는 것이 아니라 state가 공유되어야할 상황에서만 쓰는 것
- redux 및 redux-toolkit 라이브리러리로 진행
- 설치 방법: npm install @reduxjs/toolkit react-redux

>진행순서

1. store 생성<BR>src/store/index.ts


- store (state 저장소)
  - [리덕스에서 타입스크립트 쓰려면 몇가지 타입 선언을 해야함](https://redux-toolkit.js.org/tutorials/quick-start){:target="_blank"}
    ```ts
    // store 에서 선언해줘야 하는 타입들(TS)
    // root state 타입 정의
    export type RootState = ReturnType<typeof store.getState>;
    // dispatch 타입 정의, dispatch 함수의 generic type
    export type AppDispatch = typeof store.dispatch;
    ```
  - Root 파일에서 store를 import 한다.
    ```ts
    import { Provider } from "react-redux";
    import { store } from "스토어저장경로";

    <Provider store={store}>
      // 컴포넌트들
    </Provider>
    ```
  - `slice` : 리덕스 저장소(store)에 저장된 각각의 state들을 slice 라고 부른다. 하나의 state를 관리하고 처리하는 모듈이다.
    ```ts
    import { createSlice } from "@reduxjs/toolkit";
    // slice를 생성
    export const componentSlice = createSlice({
      name: "slice(state)의 이름",
      initialState: {}, // state 초기값
      reducers: {}, // state 변경함수 목록
    })
    export interface componentSlice {
      // 컴포넌트에 필요한 속성들
    }
    // 초기값이 객체이다보니 타입을 만들어야 한다.(TS)
    ```
