# 1. React ?

React는 페이스북에서 만든 자바스크립트 UI 라이브러리이다.<BR>React를 활용하면 컴포넌트(사용자 설정 태그같은 것)를 중심으로 UI를 만들 수 있다.

<BR><BR>

# 2. 개발환경 구축

Create-React-App 을 사용한 React 개발환경 구축 방법을 배웠다.<BR>
Create-React-App을 설치하기 위해서는 npm이 필요한데 npm은 Node.js로 만들어진 여러 앱을 명령어 환경에서 손쉽게 설치할 수 있게 도와주는 도구이다.<BR>(npm은 Node.js를 위한 일종의 앱스토어 같은 것) 

따라서 Node.js (npm) -> Create-React-App 순서대로 구축을 해서 React 개발 환경을 구축하면 된다.

<BR>

## 2-1. Node.js (npm)

- Chrome의 V8(JS 엔진)을 이용해서 JS 코드를 컴파일해서 실행할 수 있게 해준다.
  - JS 엔진만을 활용하므로 브라우저에 얽매이지 않고 다양한 환경에서 JS를 이용할 수 있다.
- JS 빌드와 JS 라이브러리(패키지, 모듈) 관리를 도와주는 프로그램이다.

https://nodejs.org/ko/

[공식 홈페이지](https://nodejs.org/ko/){:target=_blank}에서 Node.js를 다운로드 받고 설치를 진행한다.

cmd 창에<BR>
`node --version`<BR>
`npm --version`<BR>
을 입력하여 잘 설치되었는지 확인해보면 된다.

<BR>

## 2-2. npm(node package manager)

cmd 창에서 명령어로 원하는 모듈(특정 기능을 하는 JS 소스코드들의 모음)을 받는 방법은 다음과 같다.

- `npm install 모듈명(라이브러리이름)`
  - 해당 모듈을 npm 저장소(npmjs)에서 받아와서 설치한다.
- `-g` (global, 전역)를 기입하면 어디서나 해당 모듈을 사용할 수 있다.<BR>`-g` 옵션이 없다면 현재 폴더에 서만 이용 가능한 모듈이 설치된다.
- `npm start`
  - 개발 모드로 프로젝트를 실행한다. (코드를 수정할 때마다 서버를 재기동)
- `npm run build`
  - 배포용으로 프로젝트를 빌드(소스코드 여러개를 컴파일 연결, 링킹)한다.

<BR>

## 2-3. Create-React-App

create-react-app 을 `npx` 혹은 `npm` 키워드를 이용해 구축해준다.<BR>npm은 컴퓨터 내에 설치를 하는 것이고 npx는 임시로 설치해서 실행할때마다 설치를 하고 이용하지 않을때 삭제하는 방법이다.

> JS 로 React를 다룰 때

- `npm install -g create-react-app`
  - 한번 다운로드 받고 이후 추가적인 다운로드가 필요 없다.
  - 최신 버전을 사용하지 못할 수 있다.
- `npx create-react-app`
  - 매번 새롭게 다운로드 받아야 한다. (다운로드 이후 삭제됨, 용량차지 x)
  - 항상 최신버전의 react를 사용할 수 있다.

> TS 로 React를 다룰 때

- `npx create-react-app 현재경로폴더 --template typescript`
  - Javascript 대신 Typescript 탬플릿으로 받는다.


<BR><BR>

# 3. React 실행 구조

1. /public/index.html을 로딩한다.
2. /src/index.tsx를 로딩한다.
3. App 컴포넌트(최상위컴포넌트)를 id가 root인 요소에 렌더링한다.

# 3-1. 주요 폴더 및 파일

- `/node_modules` : npm install 로 설치한 JS 라이브러리들이 위치
- `package.json` : 앱 설명, 사용하는 라이브러리 목록, 실행 스크립트
- 
WebPack - 빌드도구
 - 프로젝트 구조에 따라서 코드를 결합하고 연결해주는 도구
