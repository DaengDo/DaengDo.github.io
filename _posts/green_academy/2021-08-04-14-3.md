# 1. 함수 (function)

- 특정한 기능을 수행하는 코드 블록이다.
- 유사한 동작을 하는 곳에서 재사용성이 좋다.
- 코드가 길 때 코드를 분리하는 용도로도 쓴다.

<BR>

# 2. 함수 선언(function declaration)

```js
function name ( parameters ) {
  ...body...
}
```

`function` 예약어(keyword), 함수이름(name), 괄호로 둘러싼 매개변수(parameters)를 차례로 적으면 함수를 선언할 수 있다.<BR>만약 매개변수가 여러 개 있다면 각 매개변수를 콤마로 구분해준다.<BR>그리고 함수를 구성하는 코드의 모임인 실행구문(body)을 중괄호로 감싸 붙여준다.

새롭게 정의한 함수는 이름 옆에 괄호를 붙여 호출(실행)할 수 있다.

```js
function DaengDo() {
  alert( "그는 신인가?" );
}

DaengDo();    // 그는 신인가?
DaengDo();    // 그는 신인가?
```

<BR><BR>

# 3. 지역 변수 (local variable)

함수 내에서 선언한 변수인 지역 변수(local variable)는 함수 안에서만 접근 할 수 있다.<BR>(중괄호 안에서 선언된 것이니 당연..)

```js
function DaengDo() {
  let message = "그는 신인가?";   // 지역 변수
  alert( message );
}

DaengDo();    // 그는 신인가?

alert( message );    // ReferenceError: message is not defined
```

<BR>

# 4. 외부 변수 (outer variable)

함수 내부에서 함수 외부의 변수인 외부 변수(outer variable)에 접근할 수 있다.<BR>(중괄호 안에서 밖의 것을 쓰는것이니 당연;;)

```js
let userName = "DaengDo";

function DaengDo() {
  let message = userName + "(은)는 신인가?"; // 외부 변수 사용
  alert( message );
}

DaengDo();   // DaengDo(은)는 신인가?
```

<BR>

# 5. 매개변수 (parameter)

매개변수(parameter)를 이용해 임의의 데이터를 함수 안에 전달할 수 있다.

```js
function DaengDo( who, what ) {
  let message = who + "(은)는 " + what + "인가?";
  alert( message );
}

DaengDo( "당신", "무엇" );      // 당신(은)는 무엇인가?
```

<BR>

# 6. 기본값 (default value)

매개변수에 값을 전달하지 않으면 그 값은 `undefined`가 된다.

위의 매개변수 예시에서 `DaengDo(who, what)` 함수는 매개변수가 2개지만, 하나 혹은 넣지않아도 함수를 호출할 수 있다.
```js
DaengDo( "너" );        // 너(은)는 undefined인가?
DaengDo();       // undefined(은)는 undefined인가?
```

매개변수에 값을 전달하지 않아도 그 값이 `undefined`가 되지 않게 하려면 '기본값(default value)' 을 설정해주면 된다.<BR>매개변수 오른쪽에 `=`연산자를 붙이고 `undefined` 대신 설정하고자 하는 기본값을 설정하면 된다.

```js
function DaengDo( who, what = "무엇?" ) {
  let message = who + "(은)는 " + what + "인가?";
  alert( message );
}

DaengDo( "너" );       // 너(은)는 무엇?인가?
```

이제 `what` 이 값을 전달받지 못해도 `undefined` 대신 기본값 `무엇?`이 할당된다.

<BR>

## 6-1. 매개변수 기본값을 설정하는 또 다른 방법

함수의 body에 기본값을 설정할수도 있다.
```js
function DaengDo( who, what ) {
  if ( what === undefined ) {
    what = "(빈 문자열!)";
  }
  let message = who + "(은)는 " + what + "인가?";
  alert( message );
}

DaengDo( "너" );    // 너(은)는 (빈 문자열!) 인가?
```
<BR>

`if`문 사용도 가능하며, 논리 연산자 `||` 사용도 가능하다.
```js
function DaengDo ( who, what ) {
  what = what || "(빈 문자열!!)";
  let message = who + "(은)는 " + what + "인가?";
  alert( message );
}
```
<BR>

이 외에도 지난 시간에 배운 'null 병합 연산자' `??` 를 사용하면 `0` 처럼 불린값 false 로 평가되는 값들을 일반 값처럼 처리할 수 있어서 편리하다.

```js
// 매개변수 'count'가 넘어오지 않으면 'unknown'을 출력해주는 함수
function showCount(count) {
  alert(count ?? "unknown");
}

showCount(0); // 0
showCount(null); // unknown
showCount(); // unknown
```

# 7. 반환 값 (return)

JS역시 함수에 반환 값(retrun)을 지정할 수 있다.
```js
function sum( a, b ) {
  return a + b;
}

let result = sum( 1, 2 );
alert( result );      // 3
```

<BR>

`return`은 함수 내 어디서든 사용할 수 있다. 실행 흐름이 `return`을 만나면 함수 실행은 즉시 중단되고 그 값을 반환한다.

함수 하나에 여러개의 `return`문이 올 수도 있다.

```js
function checkAge( age ) {
  if ( age >= 18 ) {
    return true;
  } else {
    return confirm( "보호자의 동의를 받았나요?" );
  }
}

let age = prompt( "나이를 적으세요", 18 );

if ( checkAge(age) ) {
  alert( "접속 허용" );
} else {
  alert( "접속 불가" );
}
```
<BR>

> 주의사항
- `return` 문이 없거나 `return` 지시자만 있는 함수는 `undefined`를 반환한다.

# 8. 함수 이름짓기

함수는 어떤 동작을 수행하기 위한 코드를 모아놓은 것이다.
따라서 함수의 이름은 대게 동사다.

- 함수의 이름은 가능한 한 간결하고 명확해야한다.
- 함수 이름만 보고도 함수가 어떤 기능을 하는지 힌트를 얻을 수 있어야 한다.
- 동사를 접두어로 붙여 함수 이름을 만드는 관습이 있다. (아래는 예시)
  - `show...`: ...무언가를 보여줌
  - `get...` : ...값을 반환함
  - `calc...`: ...무언가를 계산함
  - `creat...`: ...무언가를 생성함
  - `check...`: ...무언가를 확인하고 불린값을 반환함
