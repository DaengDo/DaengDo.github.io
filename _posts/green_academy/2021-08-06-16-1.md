# 객체
- JS의 자료형 중 하나이다.
  - JS의 다른 원시형자료와 다르게 여러 데이터를 담을 수 있다.
- 관련된 데이터와 함수의 집합체이다.(객체 안에 있을 때는 보통 프로퍼티와 메소드라고 부른다.) 

## 1. 리터럴과 프로퍼티

- 중괄호 `{...}` 안에는 '키:값' 쌓으로 구성된 프로퍼티가 들어간다.
```js
let user = {       // 객체
  name: "DaengDo", // 키: "name", 값:"DaengDo"
  age: 27          // 키: "age", 값: 30
};
```
  - `:` 콜론 기준으로 왼쪽엔 키가 오른쪽엔 값이 위치한다.
  - 프로퍼티 키는 프로퍼티 '이름' 혹은 '식별자' 라고도 부른다.
- 점 표기법(dot.notation)을 이용하면 프로퍼티 값을 읽는 것도 가능하다.
```js
// 프로퍼티 값 얻기
console.log(user.name); // DaengDo
console.log(user.age);  // 27
```
- 프로퍼티 값에는 모든 자료형이 올 수 있다.
```js
// 불린형 또한 추가가 가능한 모습
user.isAdmin = true;
```
- `delete` 연산자를 사용하면 프로퍼티를 삭제할 수 있다.
```js
delete user.age;
```
- 여러 단어를 조합해 프로퍼티 이름을 만든 경우엔 프로퍼티 이름을 따옴표로 묶어줘야 한다.
```js
let user = {
  name: "DaengDo",
  age: 27,
  "like games": true  // 복수의 단어는 따옴표로
};
```
- 마지막 프로퍼티 끝은 쉼표로 끌날 수 있다.
```js
let user = {
  name: "DaengDo",
  age: 27,
  "like games": true,
};
```
  - 해당 쉼표를 'trailing(길게 늘어지는)' 혹은 'hanging(매달리는)' 쉼표라고 부른다.<BR>모든 프로퍼티가 유사한 형태를 보여서 프로퍼티를 추가, 삭제, 이동하는게 쉬워진다.

<BR>

> 상수 객체는 수정될 수 있다.

```js
const user = {
  name: "DaengDo",
};

user.name = "뎅도";

console.log(user.name);   // 뎅도
```
  - `const`에 `user` 주소를 고정하는 것이지, 그 내용을 고정하는 것이 아니다.
  - `const`는 `user = ...`를 전체적으로 설정하려고 할 때만 오류가 발생한다.

<BR><BR>

## 2. 대괄호 표기법

- 여러 단어를 조합해 프로퍼티 키를 만든 경우엔, 점 표기법을 사용해 프로퍼티 값을 읽을 수 없다.
  ```js
  // 문법 에러 발생
  user.like games = true
  ```
  - 점 표기법(dot notation)은 '유효한 변수 식별자'인 경우에만 작동한다.<BR>'유효한 변수 식별자'에는 공백, 숫자로 시작하지 않기, `$`와 `_`를 제외한 특수 문자가 없어야 한다.
- 키가 '유효한 변수 식별자'가 아닌 경우에는 점 표기법 대신에 '대괄호 표기법(square bracket notation)'이라 불리는 방법을 사용한다.<BR>대괄호 표기법은 키에 어떤 문자열이 있던 상관없이 동작한다.
  ```js
  let user = {};

  // set
  user["like games"] = true;

  // get
  console.log(user["like games"]); // true

  // delete
  delete user["like games"];
  ```
  - 대괄호 표기법 안에서 문자열을 따옴표로 묶어줘야 한다.

- 대괄호 표기법을 사용하면 문자열뿐만 아니라 모든 표현식의 평가 결과를 프로퍼티 키로 사용할 수 있다.
  ```js
  let key = "like games";

  user[key] = true;
  ```
  - 변수 `key`는 런타임에 평가되기 때문에 사용자 입력값 변경 등에 따라 값이 변경될 수 있다.
    ```js
    let user = {
      name: "DaengDo",
      age: 27,
    };

    let key = prompt("출력할 사용자의 정보를 입력하세요.", "name");

    alert(user[key]); // DaengDo (프롬프트창에 name을 입력한 경우)
    ```
  - 반면 점 표기법은 이것이 불가능하다.
    ```js
    let user = {
      name: "DaengDo",
      age: 27,
    };

    let key = prompt("출력할 사용자의 정보를 입력하세요.", "name");

    alert(user.key); // undefined
    ```
<BR>

> 계산된 프로퍼티(computed property)

- 객체를 만들 때 객체 리터럴 안의 프로퍼티 키가 대괄호로 둘러싸여 있는 경우, 이를 계산된 프로퍼티(computed property) 라고 부른다.
  ```js
  let fruit = prompt("어떤 과일을 구매하시겠습니까?", "apple");

  let bag = {
    [fruit]: 5, // 변수 fruit에서 프로퍼티 이름을 동적으로 받아온다.
  };

  alert( bag.apple ); // fruit에 "apple"이 할당되었다면, 5가 출력된다.
  ```
    - 표현식이 올 수도 있다.
  ```js
  let fruit = 'apple';
  let bag = {
    [fruit + 'Computers']: 5 // bag.appleComputers = 5
  };
  ```

## 3. 단축 프로퍼티

- 프로퍼티 값을 기존 변수에서 받아와 사용해야 하는 필요성이 있는 경우 (아래 예시)
  ```js
  function makeUser(name, age) {
    return {
      name: name,
      age: age,
      // ...etc
    };
  }
  let user = makeUser("DaengDo", 27);
  alert(user.name); // DaengDo
  ```
- 프로퍼티 값 단축 구문(property value shorthand)을 사용하면 코드를 짧게 줄일 수 있다.<BR> `name: name` 대신에 `name`만 적어줘도 프로퍼티를 설정할 수 있다.
  ```js
  function makeUser(name, age) {
    return {
      name,   // name: name 과 같음
      age,    // age: age 와 같음
    }
  }
  ```
- 일반 프로퍼티와 단축 프로퍼티를 함께 사용하는 것도 가능하다.
  ```js
  let user = {
    name,   // name: name 과 같다.
    age: 27
  };
  ```

## 4. 프로퍼티 이름의 제약사항

- 객체 프로퍼티에는 예약어를 사용해도 제약이 없다.

```js
let obj = {
  for: 1,
  let: 2,
  while: 3,
  return: 4,
  function: 5,
}
console.log(obj.function);
```

- 어떤 문자형, 심볼형 값도 프로퍼티 키가 될 수 있다.<BR>문자형이나 심볼형에 속하지 않은 값은 문자형으로 자동 형 변환 된다.(심볼형 추가 공부 필요)

```js
let obj = {
  0: "test"   // "0": "test" 와 동일하다.
};
console.log(obj["0"]); // test
console.log(obj[0]);   // test
```

- 하지만 `__proto__` 는 쓸 수 없다.

```js
let obj = {
  __proto__: 5,
};

console.log(obj.__proto__); // 5를 할당했지만 값은 객체가 뜬다.
```

## 5. 'in' 연산자로 프로퍼티 존재 여부 확인하기

- 다른 언어와 다르게 JS 객체는 존재하지 않는 프로퍼티에 접근하려 해도 에러가 발생하지 않고 `undefined`를 반환한다.
  ```js
  let user = {};
  console.log(user.name === undefined); // true
  ```
- `undefined`와 비교하는 것 이외에도 연산자 `in`을 사용하면 프로퍼티 존재 여부를 확인할 수 있다.
  ```js
  // 문법 형식
  "key" in object

  // 예시
  let user = {name:"DaengDo", age:27,};

  console.log("name" in user); // user.name이 존재하므로 true 출력
  console.log("korean" in user); // user.korean이 존재하지 않으므로 false 출력 
  ```
- 프로퍼티 존재 여부를 알아보는 방법으로 `(=== undefined)`가 실패하는 경우가 있다. 프로퍼티는 존재하는데 값에 `undefined`를 할당하는 경우이다.
  ```js
  let obj = {
    test: undefined,
  };

  console.log(obj.test === undefined) 
  // test 프로퍼티는 존재하나, 값이 undefined 라서 true가 출력된다.
  console.log("test" in obj); // in 을 사용하면 프로퍼티 유무를 제대로 확인할 수 있다.
  ```

## 6. 'for...in' 반복문

- `for..in` 반복문을 사용하면 객체의 모든 키를 순회할 수 있다.

```js
/* 'for...in' 문법 */
for (key in object){
  // 각 프로퍼티 키(key)를 이용하여 본문(body)을 실행한다.
}


/* 예시 */

let user = {
  name: "DaengDo",
  age: 27,
  isAdmin: true,
};

for(let key in user) {
  // 키
  console.log(key); // name, age, isAdmin
  // 키에 해당하는 값
  console.log(user[key]); // DaengDo, 27, true
}
```
- `for...in` 반복문에서도 `for(;;)` 반복문 처럼 반복 변수를 선언(`let key`)한다.<BR>반복 변수명은 `key`말고 다른 변수명을 자유롭게 사용해도 된다.

<BR>

> 객체 정렬 방식
- 객체의 프로퍼티는 규칙에 따라 순서가 정해진다.
- 정수 프로퍼티는 자동으로 정렬되고, 그 외의 프로퍼티는 객체에 추가한 순서 그대로 정렬된다.

```js
let codes = {
  "49": "독일",
  "41": "스위스",
  "44": "영국",
  // ..,
  "1": "미국"
};

for (let code in codes) {
  console.log(code); // 1, 41, 44, 49
}
```
  - 정수 프로퍼티이기 때문에 항상 순서대로 정렬된다.<BR>`Math.trunc(Number(숫자))` 를 했을때 기존에 입력한 값과 같으면 정수 프로퍼티이다.

```js
// 번호가 정수로 취급되지 않도록 하면 된다.
let codes = {
  "+49": "독일",
  "+41": "스위스",
  "+44": "영국",
  // ..,
  "+1": "미국"
};

for (let code in codes) {
  alert(code); // 49, 41, 44, 1
}
```
