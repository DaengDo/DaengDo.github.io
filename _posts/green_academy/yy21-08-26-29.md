CBD (component based design) (90~2000년)

Container 컴포넌트와 Presentational 컴포넌트 로 분리해서 개발하기도 한다.

- 재사용
- 코드가 길어져서 가독성이 떨어짐 (생산성의 문제)
  -> 역할(role)별로 분리
    - UI 처리 영역(Presenter) : template(출력구조)
    - 로직 처리 영역(display logic) : display logic(출력흐름처리)
      : Container -> Presenter (props-down, data-down)
      : Presenter -> Container (events-up, action-up)
