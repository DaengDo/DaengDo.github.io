React를 Typescript로 작성하기 위해서 JS와 다른점 위주로 살펴보았다.

<BR>

# 1. 자료형(타입)을 명시해야함

```ts
let str: string;
let num: number;

str = "DaengDo";
// str = 1;       // type error

num = 1;
// num= "DaengDo" // type error
```
- `변수: 타입`
  - 변수의 자료형(data type)을 작성해 줘야한다.

```ts
// Typescript 코드
function sum(a: number, b: number): number {
  return a + b;
}
console.log(sum(1, 2));
```

- `function 함수명 (매개변수1: 타입, 매개변수2: 타입): 함수의반환타입`
  - 매개변수의 타입과 함수의 반환 타입을 작성해야 한다.

## 1-1. 타입 명시

- `string`
- `number`
- `[]` : 배열
  - `number[]` : 숫자 배열
  - `string[]` : 문자 배열

(계속 공부하며 추가 예정)

<BR><BR>

# 2. 타입 추론

```ts
let name = "DaengDo";
console.log(name.toUpperCase());       // DAENGDO

// name = 1;      // type error

function capitialize(str: string) {
  return str[0].toUpperCase() + str.substr(1);
}
console.log(capitalize("typescript")); // Typescipt
```
변수 `name`과 함수 `capitalize`의 반환타입을 지정하지 않은 예시이지만 정상 작동하는 TS 코드이다.

TS는 위처럼 타입을 지정하지 않아도 첫 번째 대입값에 따라서 형식을 자동으로 지정해준다.<BR>이 기능을 '타입추론(Type Inference)'이라고 한다.

<BR><BR>

# 3. interface 객체

- TS에서 객체를 다루려면 인터페이스(객체구조의 형식)를 만들어야 한다.
 
```ts
interface User {
  firstname: string;
  lastname?: string; 
  phone?: number;
}

// interface 타입명 {
//   속성명: 타입;
//   속성명: 타입;
// }
```
- `속성명?`
  - optional(필수값이 아닌) 속성으로 취급된다.


<BR><BR>

# 5. generic (타입 매개변수)

```ts
function identity<Type>(arg: Type):Type {
  return arg;
}

let output1 = identity<string>("Typescript");
let output2 = identity<number>(1);
```

- `<Type>` : 함수의 타입이 매개변수로 올 수 있다.
- 다양한 타입에 따라서 실행 처리를 다르게 할 수 있다.<BR>(내부 코드를 분기할 수 있다.)

<BR><BR>

# 6. any

TS에서 JS처럼 동적 타입이 필요할 때 사용하는 자료형이다.

```ts
const obj: any = {};
obj.name = "hong";
obj["phone"] = "01012345678";
delete obj.name;

let var1: any;
var1 = "abc";
var1 = 123;
```
- `any` : TS에서 JS처럼 자유롭게 동적 타입을 활용할 수 있다.
- 동적 타입이 필요할 경우 JS로 작성을 하면 되기 때문에 제한적인 상황에서 용도가 있다.
