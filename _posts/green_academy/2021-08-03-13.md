# 1. 비교 연산자

비교 연산자는 불린 값을 반환한다.

- `>` : 보다 큼
  - `>=` : 크거나 같음
- `<` : 보다 작음
  - `<=` : 작거나 같음
- `==` : 같음(동등), assignment operator(`=`)와 다르다.
- `!=` : 같지 않음(부등), '≠' 를 나타낸다.
<BR>

## 1-1. 불린형 반환

- 비교 연산자의 반환 값은 불린형이다. (`true` 혹은 `false`)
  ```js
  console.log( 2 > 1);    // true
  console.log( 2 == 1);  // false
  console.log( 2 != 1);   // true
  ```
- 반환된 불린값은 다른 여타 값처럼 변수에 할당 할 수 있다.
  ```js
  let result = 5 > 4;      // true 값 변수 할당
  alert( result );            // true
  ```
<BR>

## 1-2. 문자형 비교

- 문자 비교 시 문자에 할당된 유니코드 값(숫자)의 크기를 비교한다.<BR>([유니코드 표](https://unicode-table.com/kr/blocks/basic-latin/){:target="_blank"} 기준)
  ```js
  console.log('Z' > 'A'); // 005A > 0041 로 true 반환
  console.log('a' > 'A'); // 0061 > 0041 로 treu 반환
  ```
- 문자열 비교 시 적용되는 알고리듬은 다음과 같다.
  1. 두 문자열의 첫 글자를 비교한다.
  2. 첫 번째 문자열의 첫 글자가 다른 문자열의 첫 클자보다 크면(작으면), 첫 번째 문자열이 두 번째 문자열보다 크다고(작다고) 결론 내고 비교를 종료한다.
  3. 두 문자열의 첫 글자가 같으면 두 번째 글자를 같은 방식으로 비교한다.
  4. 글자 간 비교가 끝날 때까지 이 과정을 반복한다.
  5. 비교가 종료되었고 문자열의 길이도 같다면 두 문자열은 동일하다고 결론 낸다. 비교가 종료되었지만 두 문자열의 길이가 다르면 길이가 긴 문자열이 더 크다고 결론 낸다.

  ```js
  console.log('Glow' > 'Glee');
  /*
   'G'와 'l'까지는 같다. (4번 항목)
   3번째 글자 'o'와 'e'를 비교한다. 
   006F > 0065, 따라서 true
  */
  
  console.log("Bee" > "Be");
  // 5에 해당, true
  ```

<BR>

## 1-3. 다른 형을 가진 값 간의 비교

- 비교하려는 값의 자료형이 다르면 JS에서는 이 값들을 숫자형으로 바꾼다.
```js
console.log('2' > 1); // true, 문자열 '2'가 숫자 2로 바뀜
console.log('01' == 1); // true, 문자열 '01'이 숫자1로 바뀜
```
- 불린값의 경우 true 는 1, false 는 0 으로 변환된 후 비교가 이뤄진다.
```js
console.log(true == 1); // true
console.log(false == 0); // true
```

<BR>

## 1-4. 일치 연산자, ===

- 동등 연산자(equality operator)`==`는 0과 false를 구별하지 못한다.
```js
console.log( 0 == false ); // true
console.log( "" == false ); // true
```
  - 동등 연산자 `==`가 형이 다른 피연산자를 비교할 때 피연산자를 숫자형으로 바꾸기 때문에 발생한다.
- 일치 연산자(strict equality operator) `===`를 사용하면 형 변환 없이 값을 비교할 수 있다.
```js
console.log( 0 === false ); // false
console.log( "" === false ); // false
```
  - 자료형의 동등 여부까지 검사한다.

<BR>

## 1-5. null과 undefined의 비교

- `<`, `>`, `<=`, `>=` 로 비교시<BR>`null`은 `0`<BR>`undefined`는 `NaN` 으로 변한다.
> `==` 동등 연산자의 경우<BR>
```js
console.log ( null == undefined );  // true
```
  - `null` 과 0 을 비교시 주의해야 할 점
    ```js
    console.log( null > 0);   // false
    console.log( null == 0);  // false
    console.log( null >= 0);  // true
    ```
  - `==` 연산자는 형 변환을 하지 않는다. <BR>null 이나 undefined를 다른 값과 비교할때는 무조건 false를 반환한다.

- `===` 일치 연산자로 null과 undefined 비교시 false 가 반환된다. (자료형이 다르기 때문)
```js
console.log ( null === undefined ); // false
```
- `undefined` 를 다른 값과 비교해서는 안된다.
```js
console.log ( undefined > 0);   // false (1)
console.log ( undefined < 0);   // false (2)
console.log ( undefined == 0); // false (3)
```
  - (1)과 (2)의 경우에는 undefined가 NaN 가 된다. <BR>NaN이 피연산자인 경우 항상 false를 반환한다.
  - (3)의 경우 undefined는 null 이외의 값과는 같지 않으므로 false를 반환한다.

# 2. 조건 처리

## 2-1. if 문

if (조건문) {실행문} 의 구조로 되어있다.
- 조건문이 true 일 경우 실행문이 실행된다.

<BR>

## 2-2. 불린형으로의 변환

- `0`, `""`, `null`, `undefined`, `NaN` 은 불린형으로 변환시 모두 `false` 가 된다.<BR>(falsy)

```js
if ( 0 ) {        // 0은 불린형 변환시 false 이므로
  ...              // 이 코드 블록은 실행되지 않는다.
}
```
- 위의 경우를 제외하면 불린형으로 변환시 모두 `true`가 된다.<BR>(truthy)

```js
if ( "0" ) {      // "0" 은 문자열 이므로 true 값이다.
  ...
}
```

<BR>

## 2-3. 'else' 절

if 문에는 else 절을 붙일 수 있다.<BR>
else 뒤에 이어지는 코드 블록은 조건이 거짓일 때 실행된다.
```js
let test = prompt( "1+1 의 값은?" )

if ( test == 2 ) {
  alert( "골-든 정답입니다" );
} else {
  alert( "다시 풀어보세요!" );   // 2 이외의 값을 입력시 출력
}
```

<BR>

## 2-4. 'else if'

else 절에 조건을 추가 할 수 있다.
```js
let test = prompt( "1+1 의 값은?" )

if ( test < 2 ) {
  alert( "숫자를 올리세요!" );
} else if ( test > 2 ){   // 2보다 큰 값의 조건 추가
  alert( "숫자를 내리세요!" );
} else {
  alert( "정답입니다!!" );
}
```

<BR>

## 2-5. 조건부 연산자 '?'

- 조건부 연산자는 물음표 `?` 로 표시한다.
- 피연산자가 세 개이기 때문에 '삼항(ternary) 연산자' 라고 부르기도 한다.
- if 조건문을 아래와 같이 변형할 수 있다.
```js
let result = 조건문 ? value1 : value2;
```
  - `조건문`의 불린값이 true면 `value1`이, 그렇지 않으면 `value2`가 반환된다.
- 
```js
// 18세 이상이면 ok 미만이면 no를 출력하는 예시
let accessAllowed = ( age >= 18 ) ? "ok" : "no";
```
<BR>

## 2-6. 다중 '?'

연산자 `?` 를 여러 개 연결하면 else if 조건문을 처리할 수 있다.
```js
let age = prompt('나이를 입력해주세요.', 18);

let message = (age < 3) ? '아기야 안녕?' :
  (age < 18) ? '안녕!' :
  (age < 100) ? '환영합니다!' :
  '나이가 아주 많으시거나, 나이가 아닌 값을 입력 하셨군요!';

alert( message );
```
(강사님은 가독성이 안좋다며 꼭 필요한 경우를 제외하면 비권장하셨다)

# 3. 논리 연산자

JavaScript 에는 세 종류의 논리 연산자 `||`(OR), `&&`(AND), `!`(NOT)이 있다.

<br>

## 3-1. || (OR)

OR(논리합) 연산은 피연산자 중에 하나라도 true라면 true를 반환하며, 그외는 false로 처리한다.
```js
console.log( false || false );    // false
console.log( true || false );     // true
console.log( false || true );     // true
console.log( true || true );      // true
```

> JS 의 `||` (OR) 연산

- 왼쪽 항이 true로 판별되면 왼쪽 항을 반환한다.<BR>왼쪽 항이 false로 판별되면 오른쪽 항을 반환한다.<BR>(왼쪽 항의 true/false 여부가 중요)
  - [불린값을 참고](../../greenacademy/13/#2-2-불린형으로의-변환) ("", 0, undefined, NaN, null 은 false)
```js
console.log( "" || 0 );    // 0 반환
let a = "" || 0;
console.log( a );          // 0 반환
console.log( undefined || "b" );    // b 반환
```

<BR>

## 3-2. && (AND)

AND(논리곱) 연산은 피연산자 중에 하나라도 false라면 false를 반환하며, 그외는 true로 처리한다.
```js
console.log(false && false);  // false
console.log(true && false);   // false
console.log(false && true);   // false
console.log(true && true);    // true
```

> JS의 `&&` (AND) 연산

- 왼쪽 항이 true로 판별되면 오른쪽 항을 반환한다.<BR>왼쪽 항이 false로 판별되면 왼쪽 항을 반환한다.
  - [불린값을 참고](../../greenacademy/13/#2-2-불린형으로의-변환) ("", 0, undefined, NaN, null 은 false)
```js
console.log("" && 0);       // ""
console.log(1 && "a");     // "a"
let b = 1 && "a";
console.log(b);               // "a"
```

<BR>

## 3-3. ! (NOT)

논리 연산자 NOT은 느낌표 `!`를 써서 만들 수 있다.

NOT 연산자는 인수를 하나만 받고, 다음 순서대로 연산을 수행한다.
1. 피연산자를 불린형(true/false)으로 변환한다.
2. 변환된 값의 역을 반환한다.

```js
let a = true;          // true

console.log( !a );    // false
```

> `!!`

NOT 연산을 두번 붙이면 변수의 논리 값을 추출 할 수 있다.

```js
let b = "";

console.log( !b );    // true
console.log( !!b );   // false
```

<BR>

## 3-4. null 병합 연산자 '??'

null 병합 연산자 (nullish coalescing operator) 는 `??`를 사용한다.

기존의 ||(OR) 불린값 활용으로는 기본값 계열(0, "")과 빈 값(undefined, null)의 차이를 구분하기 어려웠다.<BR>
(false로 뭉뚱그리기 때문)

null 병합 연산자 `??`를 사용하면 피연산자 중 '값이 할당된' 변수를 빠르게 찾을 수 있다.

`a ?? b` 의 결과는 다음과 같다.
- `a` 가 `null` 도 아니고 `undefined` 도 아니면 `a`
- 그 외의 경우는 `b`

세 변수 중 실제 값이 있는 변수의 값을 출력하는데, 세 변수 모두 값이 없다면 '익명의 사용자' 가 출력되는 예시는 아래와 같다.
```js
let firstName = null;
let lastName = null;
let nickName = "뎅도";

// null이나 undefined가 아닌 첫 번째 피연산자
alert( firstName ?? lastName ?? nickName ?? "익명의 사용자" ); // 뎅도
```
