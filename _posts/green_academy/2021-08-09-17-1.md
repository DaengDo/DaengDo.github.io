# 1. 배열 선언

- 배열 선언 방법은 아래 두 가지 방식이 있다.
  ```js
  let arr = new Array();
  let arr = [];

  // 초기 요소를 넣어주는 것도 가능하다
  let arr = ["사과", "오랜지", "복숭아"];
  ```
- 각 배열 요소에는 0부터 시작하는 숫자(index)가 매겨져 있다.<BR>이 숫자들은 배열 내 순서를 나타낸다.
- 배열 내 특정 요소를 아래 문법으로 얻을 수 있다.
  ```js
  let fruits = ["사과", "오렌지", "복숭아"];

  alert( fruits[0] ); // 사과
  alert( fruits[1] ); // 오렌지
  alert( fruits[2] ); // 복숭아
  ```
- 위의 방법으로 요소를 수정할 수 있다.
  ```js
  fruits[0] = '포도'; 
  // 배열이 ['포도', '오렌지', '복숭아'] 로 바뀐다.
  ```
- 새로운 요소를 배열에 추가하는 것도 가능하다.
  ```js
  fruits[3] = '자두';
  // 배열이 ['포도', '오렌지', '복숭아', '자두'] 로 바뀐다.
  ```
- `length`를 사용하면 배열에 담긴 요소가 몇 개인지 알아낼 수 있다.
  ```js
  let fruits = ['포도', '오렌지', '복숭아', '자두'];

  console.log(fruits.length);   // 4 
  ```
- 배열 요소의 자료형에는 제약이 없다.
  ```js
  let arr = ['포도', {name: "DaengDo"}, true, 12, function() {console.log("배열의 요소입니다.");},]

  // 인덱스가 1인 요소(객체)의 name 프로퍼티를 출력한다.
  console.log(arr[1].name); // DaengDo

  // 인덱스가 4인 요소(함수)를 출력한다.
  arr[4]();      // 배열의 요소입니다.
  ```
- 배열의 마지막 요소는 객체와 마찬가지로 쉼표로 끝날 수 있다.<BR>(trailing 쉼표)
  ```js
  let fruits = [
    "사과",
    "오렌지",
    "자두",
  ];
  ```

# 2. pop-push 와 shift-unshift

- `push(...items)` : `items`를 배열 끝에 더해준다.
- `pop()` : 배열 끝 요소를 제거하고, 제거한 요소를 반환한다.
- `shift()` : 배열 처음 요소를 제거하고, 제거한 요소를 반환한다.
- `unshift(...items)` : `items`를 배열 처음에 더해준다.

<BR><BR>

# 3. 배열의 목적에 맞게 사용

배열은 순서가 있는 자료를 저장하는 용도로 만들어진 특수한 자료구조이다.<BR> 배열 내장 메서드들은 이런 용도에 맞게 만들어져있다.<BR> 배열을 사용할 때는 일반 객체처럼 다루게 되면 배열의 이점이 사라진다.<BR>(JS 내장 배열 메서드들이 작동을 하지 않음)<BR> 임의의 키를 사용해야 한다면 배열보다는 일반 객체가 더 적합한 자료구조이다.

> 잘못된 예시
- `arr.test = 5` 같이 숫자가 아닌 값을 프로퍼티 키로 사용하는 경우
- `arr[0]` 과 `arr[1000]`만 추가하고 그 사이에 아무런 요소가 없는 경우
- `arr[1000]`, `arr[999]` 같이 요소를 역순으로 채우는 경우

<BR><BR>

# 4. 성능

`push`와 `pop`은 빠르지만 `shift` 와 `unshift`는 느리다.

- 배열 앞에 무언가를 해주는 메서드는 배열의 모든 요소를 한 칸씩 당겨서 메모리를 할당해준다.
- 배열 끝에 무언가를 해주는 메서드는 요소를 옮기지 않으므로 요소를 이동시키는 메모리 관련 연산이 없어 실행 속도가 빠르다.

<br><BR>

# 5. 반복문

- `for (let i=0; i<arr.length; i++)` : 가장 빠른 방법이고 오래된 브라우저와도 호환된다.
- `for (let item of arr)` : 배열 요소에만 사용되는 모던한 문법이다.

>주의
- `for (let i in arr)` : 배열에서 절대 사용하면 안된다.<BR>모든 프로퍼티를 대상으로 순회하기 때문에 키가 숫자가 아닌 프로퍼티도 순회 대상에 포함된다.<BR>JS 내부적으로 객체와 함께 사용할 때 최적화되어 있어 배열에 사용하면 객체에 사용하는 것 대비 10~100배 정도 속도가 느리다.<BR>따라서 배열에서는 `for...in`을 사용하지 않는 것이 권장된다.

<BR><BR>

# 6. 'length' 프로퍼티

- `length` 프로퍼티는 배열의 길이를 나타내준다. <BR>(정확히는 배열 내 요소의 개수가 아니라 가장 큰 인덱스에 1을 더한 값)
- `length`의 값을 수동으로 증가시키면 아무 일도 일어나지 않지만, <BR><b>값을 감소시키면 배열이 잘린다.</b><BR>짧아진 배열은 다시 되돌릴 수 없다.
  ```js
  let arr = [1, 2, 3, 4, 5];

  arr.length = 2;      // 인덱스 1까지만 남기고 자른다.
  console.log(arr);    // [1, 2]

  arr.length = 5;      // 본래 길이로 되돌리려 해도
  console.log(arr[3]); // undefined, 삭제된 요소가 복구되지 않는다.
  ```

# 8. toString

`toString` 메서드를 호출하면 요소를 쉼표로 구분한 문자열이 반환된다.

```js
let arr = [1, 2, 3];

console.log(arr); // [1, 2, 3]
 
console.log(String(arr)); // 1,2,3
```
