# 1. HTTP 상태 코드

 서버쪽에서 클라이언트에게 받은 데이터를 응답하는 체계이다.
응답 코드는 1xx, 2xx, 3xx, 4xx, 5xx의 총 5개로 분류된다.

많이 사용되어 외울 필요성이 있는 코드만 메모하였다.

## 1xx : 정보전달
- 요청을 받았고, 작업을 진행 중이라는 의미이다.
  - `101` (Switching Protocols): 요청자가 서버에 프로토콜 전환을 요청했으며, 서버는 이를 승인하는 중임을 나타내는 코드다.

## 2xx : 성공
- 작업을 성공적으로 받았고, 이해했으며, 받아들여졌다는 의미이다.
  - `200`(ok): 요청이 성공했음.

  - `201`(Created): 요청이 성공했으며, 서버가 새 리소스를 작성했음.

  - `204`(No Content): 요청을 성공적으로 처리했지만 컨텐츠를 제공하지는 않음. 클라이언트가 현재 페이지에서 벗어나지 않아도 됨.


## 3xx : 리다이렉션
- 요청을 완료하기 위해서 리다이렉션(추가동작)이 필요하다는 의미이다.
  - `301`(Moved Permanetly): 요청한 페이지를 새 위치로 영구적으로 이동함.
  - `302`(See Other): 현재 서버가 다른 위치의 페이지로 요청에 응답하고 있지만, 향후 요청 시 원래 위치를 계속 사용해야 함.
  - `304`(Not Modified): 요청자에게 응답이 수정되지 않았음을 알려줌.
  - `307`(Temorary Redirect): 302와 동일하게 일시적인 컨텐츠 이동을 나타내나, HTTP 메소드의 변경을 허용하지 않음.
  - `308`(Permanent Redirect): 301와 동일하게 영구적인 컨텐츠 이동을 나타내나, HTTP 메소드의 변경을 허용하지 않음.


## 4xx : 클라이언트 오류
- 요청이 올바르지 않다는 의미이다. 브라우저에 직접 나타난다.
  - `400`(Bad Request, 잘못된 요청): 요청 자체가 잘못되었음.
  - `401`(Unauthorized): 인증이 필요한 리소스에 인증 없이 접근함. (브라우저에 어느 인증 방식을 사용할 것인지 보내야 함)
  - `403`(Forbidden): 서버가 요청을 거부함.
  - `404`(Not Found): 서버가 요청한 페이지를 찾을 수 없음.(요청하는 리소스가 없음)
  - `405`(Method Not Allowed): 요청에 지정된 방법을 사용할 수 없음.
  - `406`(Not Acceptable): 요청은 정상이나, 서버에서 받아들일 수 없는 요청임.
  - `409`(Conflict): 서버가 요청을 수행하는 중에 충돌이 발생함.
  - `413`(Payload Too Large): 요청 본문이 너무 길어 서버가 처리할 수 없음.
  - `414`(URI Too Long): 요청 URI(일반적으로 URL) 본문이 너무 길어 처리 불가능함.
  - `415`(Unsupported Media Type): 요청한 미디어 유형이 서버에서 지원하지 않는 유형임.
  - `429`(Too Many Requests): 사용자가 일정 시간 동안 너무 많은 요청을 보냄

## 5xx : 서버 오류
- 올바른 요청에 대해 서버가 응답할 수 없다는 의미이다.
  - `500`(Internal Server Error): 서버에 오류가 발생해 작업을 수행할 수 없음  
  - `501`(Not Implemented): 서버가 요청을 수행하는데 필요한 기능을 지원하지 않음
  - `502`(Bad Gateway): 게이트웨이가 연결된 서버로부터 잘못된 응답을 받음.
  - `503`(Service Temporarily Unavailable): 서비스를 일시적으로 사용할 수 없음.
  - `504`(Gateway Timeout): 게이트웨이가 연결된 서버로부터 응답을 받을 수 없음.
