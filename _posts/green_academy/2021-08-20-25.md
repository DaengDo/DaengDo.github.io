## Ref(참조 객체 생성)

입력한 내용이 출력되는 인풋박스를 만들어보며 Ref 개념을 배웠다.

```ts
const Ref변수 = useRef<Ref타입>(초기값);
// TS로 작업을 하기 때문에 타입을 명시해주었다.
```
- JSX Element를 참조하는 객체라면 JSX Element로 렌더링 되는 HTML 요소의 타입을 넣어야한다.
- `참조객체.current` 는 현재 렌더링된 HTML 요소를 뜻한다.

## React 컴포넌트가 재렌더링 되는 경우

- state 가 변경됐을 때
- props 가 변경됐을 때
- 부모 컴포넌트가 다시 렌더링 될 때
- key 속성이 변경 됐을 때
  - props, state 는 그대로 놔두고 화면을 다시 렌더링 하고싶을때 key값을 의도적으로 변경하기도 한다.

> 주의 사항
- 렌더링 주기에 따라서 1회만 일어나므로 값이 변경될때마다 렌더링이 일어나는 것이 아니다.
- 렌더링이 한번 일어날때 마다 state가 아닌 변수는 초기화가 일어난다.

<BR><BR>

## 클래스 / 함수형 컴포넌트

- Stateful Component
  - 상태관리 필요한 컴포넌트
  - 내부에 state로 특정 시점에 다시 렌더링
  - class component로 작성

- Stateless Component
  - 상태관리 필요하지 않은 컴포넌트
  - props만 받아서 렌더링하는 것
  - function component로 작성

## SPA

- 페이지 파일이 1개, index.html
- 특정 영역에 컴포넌트(js)를 로딩한다.
- 애플리케이션이 처음 로딩될 때 import한 모든 컴포넌트를 로딩한다.
  - 컴파일됐을 때 파일 크기가 커지고 초기 로딩 시간이 걸린다는 단점이 존재한다.<BR>이를 해결하기 위해 lazy loading 방식을 선택하기도 한다.

> Lazy-Loading
- 컴포넌트를 방문할 때 로딩을 하는방식이다.<BR>작동하려면 `suspense` 기능이 필요하다
- `import { lazy } from "react"`로 기능을 불러오면 된다.

```ts
// SPA을 불러올 때 로딩하는 방식
import Component from "경로";

// 컴포넌트를 방문할 때 로딩하는 방식
const Component = lazy(() => import("경로"));
```

<BR>

> Suspense
- 컴포넌트를 로딩중에 보여줄 화면을 처리한다.
- fallback 속성으로 로딩중에 보여줄 화면을 처리할 수 있다.<BR>(JSX 요소 예시)

```html
<Suspense fallback={<div>Loading...</div>}>
```
