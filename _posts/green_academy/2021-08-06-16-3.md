# 메서드?

- 객체 프로퍼티에 할당된 함수를 '메서드'라고 부른다.

<BR><BR>

## 1. 메서드 만들기

- 함수 표현식으로 함수를 만들고, 객체 프로퍼티로 함수를 할당할 수 있다.
  ```js
  let user = {
    name: "DaengDo",
    age: 28,
  }

  user.sayHi = function() {
    console.log("Hi");
  };

  user.sayHi(); // Hi
  ```
  - 선언된 함수를 메서드로 등록하는 방법도 있다.
    ```js
    let obj = {};

    function sayHi() {
      console.log("Hi");
    }

    obj.sayHi = sayHi();
    
    obj.sayHi;  // Hi
    ```

> 단축 구문

- `function`을 생략해도 메서드를 정의할 수 있다.
  ```js
  // 메서드 선언
  user = {
    sayHi: function() {
      console.log("Hi");
    },
  };

  // 아래는 단축구문
  user = {
    sayHi() {
      console.log("Hi");
    },
  };
  ```
  - 다만 이 방법은 객체 상속에서 차이가 있다고 한다. (추가 공부시 링크)

<BR><BR>

## 2. 메서드와 this

- `this`키워드를 활용하면 메서드가 객체에 저장된 정보에 접근할 수 있다.
  ```js
  let user = {
    name: "DaengDo",
    sayHi() {
      console.log("Hi, " + this.name + "\!");
      // this는 현재 객체를 나타낸다.
    },
  };

  user.sayHi();   // Hi, DaengDo!
  ```
<BR>

- `this`를 사용하지 않고 외부 변수를 참조해 객체에 접근하는 것도 가능하다.
  ```js
  let user = {
    name: "DaengDo",
    sayHi() {
      console.log("Hi, " + user.name + "\!");
      // this 대신 user 사용
    },
  };
  ```
  - 외부 변수를 사용해 객체를 참조하면 변수가 다른 값으로 덮어써졌을 때 에러가 발생할 수 있다.
    ```js
    let user = {
      name: "DaengDo",
      sayHi() {
        console.log("Hi, " + user.name + "\!");
        // Error: Cannot read property 'name' of null
      },
    };

    user = null; // user를 null로 덮어씀
    ```
    - 이 경우 `this`를 썼다면 에러가 발생하지 않았다.

<BR><BR>

## 3. 자유로운 this

JS에서 `this`는 모든 함수에서 사용할 수 있다.<BR>
<b>메서드 내부에서 </b>`this`<b> 키워드를 사용하면 객체에 접근할 수 있다.</b><BR>`this` 값은 런타임(컨텍스트에 따라 다름)에 결정된다.

즉, 동일한 함수라도 다른 객체나 함수(다른 컨텍스트)에서 호출했다면 `this`가 참조하는 값이 해당 컨텍스트가 되어 `this`가 가르키는 값이 다르다.
```js
/* 객체 생성 */
let user = {name: "Kim"};
let admin = {name: "DaengDo"};

/* 함수 선언 */
function sayHi() {
  console.log("Hi, " + this.name);
}

/* 각 객체가 동일한 함수 사용 */
user.func = sayHi;
admin.func = sayHi;

/* 그러나 this로 인해 결과값이 다름 */
user.func();  // Hi, Kim
admin.func(); // Hi, DaengDo
```

> 주의할점
- 이렇게 `this`가 런타임에 결정되면 좋은 점도 있고 나쁜 점도 있다. 함수(메서드)를 하나만 만들어 여러 객체에서 재사용할 수 있는 것은 장점이지만, 이런 유연함이 실수로 이어질 수 있다는 것이 단점이다.

<BR><BR>

## 4. this와 화살표 함수 메모

- 런타임에서 this context가 다른 경우 this가 고정이 안되는 문제점을 해결하기 위해 탄생한것이 자바스크립트의 화살표 함수이다. (런타임 환경에 따라 window 인 경우, 네트워크 인 경우 등등)
- 화살표 함수에서의 this는 전역함수로 고정이 되어 있다. 화살표 함수에서 this를 사용하면 외부에서 this값을 참조하게 된다.

- 화살표 함수 쓰면 this 고려 안해도 된다.

- context : 멀티쓰레드 없던 시절에 쓰레드 하나로 실행되는걸 분리해서 실행

- 자바스크립트는 쓰레드가 한개라서 context 로 처리함

- 일반적으로 화살표함수가 권장된다.<BR>this를 세부적으로 써야한다면 일반적인 함수 표현법을 사용한다.<BR>하지만 this 없이도 this와 관련된 객체를 직접 사용할 수 있어서 잘 안 쓴다.
