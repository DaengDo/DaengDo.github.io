# Annotation

- 컴파일러에게 문법 에러 채크하도록 정보를 제공해준다.
- 소프트웨어 개발 툴이 빌드나 배치 시 코드를 자동 생성하게 해준다.

<BR>

## Lombok 라이브러리 내장 annotation

`Lombok`은 java spring에 다양한 어노테이션을 제공해주는 라이브러리이다.

- `@Builder`
  - 해당 클래스에 빌더를 추가

- `@Data`
  - `@Getter`, `@Setter`, `@RequiredArgsConstructor`, `@ToString`, `@EqualsAndHashCode`를 한꺼번에 적용<BR>
  모든 필드를 대상으로 접근자와 설정자가 자동으로 생성

- `@NoArgsConstructor`
  - 파라미터가 없는 기본 생성자를 생성

- `@AllArgsConstructor`
  - 모든 필드 값을 파라미터로 받는 생성자를 만들어줌

<BR>

# spring 내장 annotation

- `@Autowired`
  - IoC 컨테이너에 있는 싱글턴 객체를 해당 필드에 주입(= 스프링이 관리하는 Bean을 주입받음)<BR>수업에서는 TodoController 싱글턴 객체를 test 필드에 주입하는 용도로 써봄

- `@Controller`
  - 해당 클래스를 Controller로 설정, 자동으로 스프링의 객체(Bean)로 등록됨

- `@ResponseBody`
  - 메소드에서 리턴되는 값을 View 를 통해서 출력되지 않고, HTTP Response Body에 직접 씀
  - 자바객체를 HTTP요청의 바디내용으로 매핑하여 클라이언트로 전송

- `@RestController`
  - `@Controller + @ResponseBody` 
  - 메소드의 return(반환 결과값)을 문자열(JSON) 형태로 반환
  - [`@Controller` 와 `@RestController`의 차이점](https://develop-log-sj.tistory.com/29){:target="_blank"}

- `@RequestBody`
  - Http요청의 본문(body)이 그대로 전달된다.
  - HTTP POST 요청에 대해 request body에 있는 request message에서 값을 얻어와 매칭, RequestData를 바로 Model이나 클래스로 매핑
  - json기반의 메시지를 사용하는 요청의 경우 이용

- `@ComponentScan`
  -  @Component, @Service, @Repository, @Controller 어노테이션이 부여된 Class들을 자동으로 Scan하여 Bean으로 등록해주는 역할을 하는 어노테이션

- `@RequestMapping`
  - 클래스 선언부 작성시 : 현재 클래스의 모든 메서드들의 기본적인 URL 경로
  - 메서드 선언부 작성시 : 클래스 선언부에 작성된 주소 뒤에 추가적으로 붙는 경로를 설정
  - 해당 어노테이션이 선언된 클래스의 모든 메소드가 하나의 요청에 대한 처리를 할경우 사용
    - 요청방식들을 동시에 설정 가능
    - @RequestMapping(value = "/test7", method = {RequestMethod.GET, RequestMethod.POST})

- `@PostMapping`
  - POST 요청을 하는 API의 어노테이션
  
- `@GetMapping`
  - GET 요청을 하는 API의 어노테이션
  - @RequestMapping과 유사하나 @GetMapping 은 메소드에만 적용.

- `@PutMapping`
  - PUT 요청을 하는 API의 어노테이션

- `@DeleteMapping`
  - DELETE 요청을 하는 API의 어노테이션

- `@SpringBootApplication`
  - 스프링 부트의 자동 설정, 스프링 Bean 읽기와 생성이 모두 자동으로 설정된다. 이것이 있는 위치부터 설정을 읽어가기 때문에 항상 프로젝트의 최상단에 위치해야한다.

- `@SpringBootConfiguration`
  - 스프링 부트의 설정을 나타냄. 스프링의 `@Configuration`을 대체함

- `@SpringBootTest`
  - 스프링부트 어플리케이션 테스트에 필요한 의존성을 제공해줌

- `@EnableAutoConfiguration`
  - 미리 정의되어 있는 Bean을 가져와서 등록해줌

- `@PathVariable`
  - URL 경로에 변수를 넣는 기능<BR>(null이나 공백값이 들어간다면 적용하지 말아야 하고, 깂을 받을 때 `.` 이 포함되어 있다면 `.`포함 그 뒤가 잘려서 들어온다.)

# JUnit 내장 annotation

- `@Test`
  - 해당 메소드가 테스트 대상 메소드임을 의미
- `@BeforeAll`
  - 해당 테스트 클래스를 초기화할 때 딱 한번 수행되는 메소드<BR>(static 으로 선언해야한다.)
- `@BeforeEach`
  - 해당 메소드는 테스트 메소드 실행 이전에 수행된다.
- `@AfterAll`
  - 해당 테스트 클래스 내 테스트 메소드를 모두 실행시킨 후 딱 한번 수행되는 메소드<BR>(static 으로 선언해야한다.)
- `@AfterEach`
  - 테스트 메소드 실행 이후에 실행
- `@Disabled`
  - 해당 테스트 메소드가 무시됨
