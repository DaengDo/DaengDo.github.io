## **자바 개발 과정**

java는 플랫폼에 독립적인 언어이다. 소스코드를 직접 기계어로 컴파일하여 링크하는 c/c++와 다르게 자바 컴파일러는 바이트코드인 클래스 파일(.class)을 생성하고 이 파일의 바이트 코드를 읽은 뒤 기계어로 바꾸어 실행하는것은 JVM (java virtual machine) 이다.

<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FbCHRHH%2Fbtq7sVyIHNb%2FzMRJeIqnXgHHot4wSkWFp0%2Fimg.png">

따라서 java는 해당 플렛폼에 맞는 JVM이 있다면 어느 환경이든 똑같은 소스코드를 입력해 똑같은 결과를 얻을 수 있다.

## **자바 개발 환경 구축**

[자바 다운로드 링크](https://java.com/ko/download/ie_manual.jsp?locale=ko)

[이클립스 다운로드 링크](https://www.eclipse.org/downloads/) 자바 특화 IDE, 2020-09(4.17) 버전부터는 JAVA 11을 요구한다.

자바의 실행은 **javac** 와 **java** 명령어가 필요하다. 폴더 경로에 상관없이 어디에서든 운영체제가 java 명령어를 알아들을 수 있게 해당 명령어 실행 파일을 운영체제의 환경 변수에 설정해야 한다.

레지스트리에 의존하지않는 유닉스 기반 OS는(예를들어 macOS) 환경 변수를 설정하지 않아도 된다. (확인필요)

윈도우10 기준

제어판 → 시스탬 및 보안 → 시스템 → 고급 시스템 설정 → 환경 변수 → 시스템 변수 '새로 만들기'

이후 JAVA_HOME 환경 변수를 설정하고 Path 환경 변수를 수정하면 된다.

변수 이름은 **JAVA_HOME**

변수 값은 각자 JDK가 설치된 경로를 붙여넣으면 된다.

<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FbCgUUN%2Fbtq7snbbYEA%2FrRCk87U0ZWLV0jGLxDV380%2Fimg.png">
<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FY5H64%2Fbtq7sYaMa24%2FbsDfSFRLJKz7tXCZl4nZHk%2Fimg.png">

새로 만들기를 누른 후 환경변수에 **%JAVA_HOME%\\bin** 을 입력한다.

이후 아무 윈도우 명령 프롬프트 창을 키고

<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FqnuRi%2Fbtq7wzAYfIY%2Fz8A5rV2lAMDC3UUfR4FmT1%2Fimg.png">

**javac -version**

**java -version**

명령어를 입력해 오류가 뜨지 않는다면 환경변수 설정이 완료된 것이다.

오류가 뜬다면 환경변수가 제대로 입력이 안된것이거나 컴퓨터를 재부팅하면 된다.

<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FbCHRHH%2Fbtq7sVyIHNb%2FzMRJeIqnXgHHot4wSkWFp0%2Fimg.png">

해당 구조를 복습하는 차원에서 간단히 바탕화면에 소스 파일을 만들고 실행시켜 보았다.

바탕 화면에서 메모장을 키고

```
public class  TestHello {

    public static void main(String[] args) {

        System.out.printin("Test Hello");
    }
}
```

TestHello.java로 저장해준 뒤 아무 윈도우 CLI 환경에서

<img src="https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdn%2FFYq0s%2Fbtq7qv1B36M%2FgUftZoRQIybKKIFn4hYKMk%2Fimg.png">

javac 명령어로 바이트 코드 파일을 만들고 java 명령어로 실행해 보았다.
