리덕스는 전역에서 사용하는 변화하는 데이터의 흐름을 단방향화해서 어플리케이션의 복잡도를 낮추는 역할을 한다. (전역 상태 관리 라이브러리)

- `redux`
  - JS어플리케이션들의 상태(변화하는 데이터)를 관리하는 방법중 하나

- `store`
  - `createStore`로 store를 생성
  - 어플리케이션 내의 바뀌는 데이터(state)를 넣는 곳
  - store 는 reducer를 필요로 함
  - 상태 관련
    - `getState()` : 현재 상태 값
    - `subscribe()` : 상태 변경을 감지

- `reducer`
  - reducer는 data를 변경하는 함수
  - reducer가 return하는 것은 app의 data(상태, state)
  - 유일하게 data를 바꿀 수 있는 곳이 reducer
  - 기본값으로 현재의 state 와 함께 불려짐

- `action`
  - redux에서 reducer를 부를때 쓰는 두번째 parameter 혹은 argument
  - reducer가 할 행동
  - reducer에게 action을 보내는 방법 `store.dispatch({key:value})`
  - 반드시 `type` property를 갖고있어야 함



#### 앱 내 리덕스 적용

어플리케이션 내에 data(글로벌 상태)를 적재하는 store를 생성한다. 앱 내 store는 하나뿐이어야하며 모든 전역 상태는 이곳에서 관리된다. 데이터를 변경할 수 있는 것은 reducer뿐이다. reducer가 어떻게 데이터를 변경할지 알려줘야하는데 그것은 action이 담당한다. action의 type에 따라 reducer가 다르게 작동한다. 데이터를 변경하고 싶으면 action을 reducer에 dispatch해야한다.

우선 action을 생성한다. 해당 action을 reducer에 dispatch한다. reducer는 변화한 data(상태값)을 return한다. 변화한 data가 view(사용자)와 상호작용한다.
