# `<length>`

<BR>

CSS 에서의 length 자료형은 거리 값을 나타낸다.<BR>width, height, margin, padding, border-width, font-size, text-shadow 등 다양한 속성에 사용할 수 있다.

<BR>

# 1. Absolute Length units (절대 길이 단위)

> 재미있는 사실은 물리적 측정치가 픽셀 단위로 매핑된다는 것이다.

- `px` (pixel)

  - 하나의 픽셀이다. 일반적으로 하나의 장치 픽셀을 나타내지만, 프린터 및 고해상도 화면의 경우 하나의 CSS픽셀이 여러 장치 픽셀을 [의미](https://css-tricks.com/wp-content/uploads/2013/03/ipad-mini.png){:target="_blank"}하기도 한다.

- `in `

  - 인치는 물리적 측정치지만, CSS에서의 1in는 96px로 매핑된다. 

- `pt` (point)

  - 1pt = 1/72 * 1in<BR>CSS 외부에서 유형의 크기를 조정하는 일반적인 방법으로, 주로 인쇄 매체에서 사용한다고한다(출처 불분명).

- `pc` (pica)

  - 1pc = 12pt = 1/6 * 1in

- `cm`

  - 1cm = 37.8px

- `mm`

  - 1mm = 1/10 * 1cm

- `Q`

  - 1Q = 1/40 * 1cm (밀리미터의 쿼터)

<BR><BR><BR><BR>

# 2. Relative length units (상대 길이 단위)

> 다른 요소나 부모 또는 디스플레이 크기에 비례해 크기가 조정된다.<BR>때문에 반응형 사이트 또는 사용자 지정 크기에 대응하는데 유리하다.

<BR>

### font-relative lengths (글꼴 상대 길이 단위)

> 글꼴 기준 길이는 해당 상위 요소(부모)에서 현재 유효한 글꼴의 특정 문자 또는 글꼴 속성의 크기로 값을 정의한다.

- `em`
  - 현재 글꼴 크기에 상대적이다. (아래 예시를 참고)
    <div class="notice--primary" markdown="1">
    `h1 { line-height: 1.2em }` 은 h1요소의 줄 높이가 h1 글꼴 크기보다 20% 크다는 의미이지만,<BR>
    `h1 { font-size: 1.2em }` 은 h1 요소의 글꼴 크기가 상속된 글꼴 크기보다 20% 크다는 의미이다. 
    </div>
- `rem`
  - tree-structure의 root 요소의 글꼴 크기에 상대적이다.<BR> 기준이 root인 것만을 빼면 em과 같다. <BR>(font-size와 마찬가지로 grid system에도 사용 가능)
- `ex`
  - 요소 font의 x 높이를 나타낸다. x가 영어에서 소문자 크기 표준으로 처리하기 쉬워서 이런 단위가 쓰인다. 많은 글꼴에서 1ex가 0.5em 이다.
- `ch`
  - 숫자 0 문자의 너비를 기준으로 한다. 문자 "0"의 너비를 측정하는 것이 불가능하거나 실용적이지 않은 경우 너비 0.5em에 높이 1em이라고 가정해야 한다.
- `cap` (cap height)
  - 영문 대문자의 평균 높이 값을 나타낸다.
- `ic`
  - "水" (한중일 한자 "물", U+6C34) 문자를 렌더링할 때 사용하는 글꼴에서의 너비를 나타낸다. (안쓸거같아서 그냥 공부안하고 복붙함.)
- `lh`
  - 요소가 line-height를 가지고 있는 경우, line-height 의 계산값을 절대 길이로 변환해 나타낸다.
- `rlh`
  - 루트 요소의 line-height를 절대 길이로 변환해 나타낸다. 루트 요소에서 쓰일 경우 default 값을 기준으로 한다.

<BR>

### Viewport-percentage lengths (뷰포트 백분율 길이 단위)

> Viewport ??
 - 현제 화면에 보여지고 있는 다각형(보통 직사각형)의 영역을 뜻한다.<BR>웹 브라우저에서는 현재 창에서 문서를 볼 수 있는 부분을 뜻한다.<BR>일반적으로는 <a target="_blank" href="../../assets/images/viewport.png">디바이스 화면 크기</a>라고 볼 수 있다.

- `vh` (view-height)
  - viewport 높이의 1% 크기이다.<BR>예) 15vh -> viewport 높이의 15%
- `vw` (view-width)
  - viewport 너비의 1% 크기이다.<BR>예) 10vw -> viewport 너비의 10%
- `vi`
  - 루트 요소의 inline 형식으로, initial containing block의 1% 크기이다. (보통의 경우 html 박스의 1%)<BR>(모질라에는 있지만 W3C 문서에는 없는 단위)
- `vb`
  - 루트 요소의 block 형식으로, initial containing block의 1% 크기이다. (보통의 경우 html 박스의 1%)<BR>(모질라에는 있지만 W3C 문서에는 없는 단위)
- `vmin`
  - viewport의 작은 (높이와 너비 중)치수의 1% 크기이다.
- `vmax`
  - viewport의 큰 (높이와 너비 중)치수의 1% 크기이다.
