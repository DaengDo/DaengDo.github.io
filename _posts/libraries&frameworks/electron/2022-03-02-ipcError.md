# 상황

app 내부에서 신뢰할 수 없는 컨텐츠에 접근하기 때문에 `nodeIntegration: false`를 할 수 밖에 없는 상황,<br>
renderer process (정확히는 preload script) 에서 바탕화면 경로를 갖고 오기 위해 ipc 통신이 필요함 <br>
ipc 통신을 통해 바탕화면 경로를 main process에서 renderer process로 전송해주려고 함

3가지 방법을 생각해냄
- `ipcRenderer.send()`로 요청 `ipcMain.on("channel", (e, args)=> { e.sender.send("channel2", "바탕화면 경로")})`
  - main 에서 바로 renderer로 가는 것이 아니고 renderer에서 요청하고 main에서 보내주는 방식이라 번잡해서 선택안함
- `ipcRenderer.invoke` 와 `ipcMain.handle` 사용
  - 위와 마찬가지의 이유로 선택안함
- `win.webContents.send` 사용
  - main process 에서 바로 renderer process로 전송할 수 있어 채택

<br>

# 에러 사항

```js
// main.js
mainWindow.webContents.send("desktopPath", app.getPath("desktop"));
```

- main process에서 webContents로 ipc 통신

```js
// preload script
ipcRenderer.on("desktopPath", (e, path) => {
  console.log(path);
});
```

- 해당 코드가 renderer proces에서 작동을 하지 않음

<br>

# 해결

```js
// main.js
mainWindow.webContents.on("did-finish-load", () => {
  mainWindow.webContents.send("path", app.getPath("desktop"));
});
```

webContents가 로딩되기전에 send를 해버려서 일어난 문제였음.

추가) preload script는 BrowserWindow가 준비되고 webContents 가 켜지기전에 동작함
