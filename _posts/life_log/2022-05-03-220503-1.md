# 자바스크립트 너무 느슨한거 아니야?

어떤 값이 특정 범위 내에 속하는지 구하는 표현식이 필요했다. 내가 짠 코드는 한 눈에 보이지 않아 코드숨 디스코드 커뮤니티에 혹시 조언을 구할 수 있을까 하여 질문을 올렸다.

```js
// a가 1 혹은 2 혹은 3 일 때
  if(a === 1 || a === 2 || a ===3){
    // 무언가 하고 싶어요. 다만, 위의 조건이 더럽네요
  }
```

곧바로 답장이 날라왔다.

```js
// 조건을 이걸로 바꿔보세요
[1, 2, 3].includes(a)
```

와우..! 좋다! 배열 메서드를 사용해 선언형 스타일로 보기 좋게 만들었다. 이건 영어 단어만 알아도 무슨 뜻인지 알겠다.

하지만 문제는 이 뒤에 있었다. 만약 이 문제가 없었다면 내 일기에 오를 일도 없었을 것 이다.

`a` 값이 문자열이었던 것

하아.... 알 수 없는 동작으로 인해 if 문을 통과하지를 못했고 이유를 찾느라 여기저기 들쑤셨다. 나중에 유의깊게 개발자 도구를 통해 찍힌 로그를 보고 나서야 깨달았다.

아뿔싸 `a` 값이 `1`이 아니라 `"1"` 이었구나 😭

타입스크립트를 왜 쓰는지 몸으로 깨달은 순간이었다. 타입채크만 제대로 했어도 이런 사태가 일어날 일이 없었는데 말이다.

리터럴로 넣은 값이 종종 문제를 일으키는 것 같다. 상수를 사용할 때 특히 유의해야겠다. `enum` 느낌으로 동결 객체에 프로퍼티로 상수 관리를 하고 있었는데 그러다보니 프로퍼티만 작성해 값을 보지 않은 탓 이었다.

값을 사용할 때에는 그 값이 정확히 어떤 값인지 파악해야겠다.

그나저나 요즘 점심시간마다 이펙티브 타입스크립트를 틈틈히 보고있는데 빨리 타입스크립트 환경으로 넘어가야겠다 ㅠㅠ
