# Fact (무슨 일이 있었나?, 사실)

5월 1일부터 코드숨이라는 코스를 신청해 참가하게 되었다.<BR>
해당 코스는 과제(아웃풋) 중심의 학습을 진행하며 학습자가 피드백을 통해 성장하게끔 유도를 한다.<BR>
평일에 틈틈히 과제를 작성해 리뷰를 받았고, 다른 사람들의 코드와 리뷰 또한 보고 배우게 되었다.<BR>
E2E 테스트 개념을 처음 알게되었다.

# Feeling (무슨 느낌이 들었나?, 느낌)

PR을 하고 어떤 리뷰가 올지 기대되고 설레었다. 마치 편지를 보내고 답장을 기다리는 느낌!

# Finding (어떤 인사이트를 얻었나?, 배운점)

답장을 바라는 사항이 있는 경우 명시해야 한다.
기록을 남기지 않으면 기억이 희석된다.

# Future action (앞으로 무엇을 할까?, 향후 행동)

작성해둔 E2E 테스트를 케이스별로 돌려볼 때 정말 편리해서 꼭 내 것으로 익히고 싶어졌다!
그동안 내가 만든것에 테스트를 적용하면서 연마하는 중!
잊고 싶지 않은 기억은 기록을 해둬야겠다. 짧게 나마 기록을 해두면 나중에 기억이 희미해지더라도 기록을 통해 복각할 수 있다.

# Feedback (어떤 피드백을 받았나?, 고칠 점)

코드 리뷰중 받았던 피드백
- 자명한 경우가 아니라면 줄임말은 사용하지 말 것
- RORO (receive object & return object), 객체를 이용해서 함수를 활용할 것
