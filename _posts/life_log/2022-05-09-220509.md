오늘은 `매개변수를 객체로 만들면 얻을 수 있는 이점` 리마인드!!
코드숨 1주차 과제 중 함수의 매개변수를 원시타입 그대로 넣어서 사용중이었습니다.

>before
```js
render(1, null, null, false)
```
내가 봐도 뭘 하고싶은건지 모르겠습니다 🤣

>after
```js
render({
  number: 1,
  operand: null,
  operator: null,
  isFinished: false,
})
```
위보다 훨씬 볼만하네요! 

어떤 이점이 생겼는지 자세히 알아보겠습니다.

# 함수의 매개 변수를 객체로 받으면 얻는 이점

### 1. 파라미터에 이름을 표현해줄 수 있다.

```js
printStatus('daengdo', '서울', true, null);
```

각 파라미터가 도대체 뭘 나타내는지 알 수 없습니다. 하지만 객체를 활용하면 프로퍼티로 각 파라미터가 무엇을 나타내는지 표현이 가능합니다.<BR>

```js
printStatus({
  name: 'daengdo',
  adress: '서울',
  isHungry: true,
  girlFriend: null
});
```
이제 무엇을 의미하는지 알았습니다. 이름은 뎅도에, 주소지는 서울, 배고프고, 여친이 없군요!

😥

### 2. 옵셔널 데이터를 다루기 쉬워진다.

만약 위의 예시에서 주소지 값이 필수가 아니라면 아래와 같이 호출해야 합니다.

```js
printStatus('daengdo', '', true, null);
```

기본 값을 줘야하고 어디에 빈 값을 줘야하는지도 알고 있어야 합니다. 하지만 객체를 이용한다면 해당 속성을 안넘기면 됩니다.

```js
printStatus({
  name: 'daengdo',
  isHungry: true,
  girlFriend: null
});
```

### 3. 순서에 상관없이 파라미터를 입력할 수 있다.

각 파라미터의 순서에 맞춰 넘겨주지 않고 이름으로 넘겨주기 때문에 순서를 신경쓰지 않아도 됩니다.

```js
printStatus('daengdo', '서울', true, null);

printStatus({
  adress: '서울',
  girlFriend: null,
  isHungry: false,
  name: 'daengdo'
}); // 상관없음!
```

### 4. 파라미터의 갯수가 늘어나도 괜찮다.

객체로 분리할 경우 파라미터가 늘어나도 짧게 표현할 수 있습니다.

```js
printStatus('daengdo', 28,'서울', true, null); // 점점 길어진다...

const person = {
  name: 'daengdo',
  age: 28, // 추가!
  adress: '서울',
  isHungry: true,
  girlFriend: null,
};

printStatus(person);  // 짧아서 좋아요 호호호오홍홍
```

### 5. 객체로 의미있는 단위로 묶을 수 있다.

비슷한 속성을 가진 파라미터들은 객체로 묶어서 의미를 전달할 수 있습니다.

```js
const user = {
  name: 'daengdo',
  age: 28, // 추가!
  adress: '서울',
  girlFriend: null,
};

const attendance = {
  absentCount: 12, 
  attendanceCount: 30,
  isPass() {
    return this.attendanceCount > 20;
  },
};

printStatus(user, attendance);
```

## 언제 사용하면 좋나요?

- 매개변수끼리 관계가 있어 구조로 묶을 수 있을 때
- 매개변수가 많아져서 복잡성이 증가할 때

🚀 앞으로는 매개변수를 많이 사용할 때는 객체를 적극 활용합시다!
