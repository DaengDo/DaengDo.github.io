- 원시 타입과 객체(참조) 타입의 다른 점 3가지
  - 원시 타입의 값은 변경 불가능한 값, 객체 타입의 값은 변경 가능한 값
  - 원시 값을 변수에 할당시 실제 값 저장, 객체를 변수에 할당시 참조 값 저장
  - 원시 값을 갖는 변수를 다른 변수에 할당시 원시 값이 복사되어 전달(값에 의한 전달, pass by value)<br>객체를 가리키는 변수를 다른 변수에 할당시 원본의 참조 값이 복사되어 전달(참조에 의한 전달, pass by reference)

# 1. 원시 값

## 1-1. 변경 불가능한 값

변경 불가능하다는 의미는 읽기 전용 값이라는 의미이다. 따라서 데이터 신뢰성을 보장한다.<br>
원시 값은 새로운 원시 값을 재할당했을 때 변수가 가리키던 메모리 공간의 주소를 바꿀 필요없이 원시 값 자체를 변경한다. 이러한 특성을 불변성이라 한다.

원시 값을 할당한 변수는 재할당 이외에 변ㄴ수 값을 변경할 수 있는 방법이 없다.

## 1-2. 문자열과 불변성

- JS에서는 문자 뿐만 아니라 문자열 또한 원시 타입이다.
- 문자열은 유사 배열 객체이며 이터러블하기 때문에 배열과 유사하게 각 문자에 접근할 수 있다.
  ```js
  var str = 'string';

  console.log(str[0]); // s
  ```

## 1-3. 값에 의한 전달

변수에 원시 값을 갖는 변수를 할당하면 할당받는 변수에는 할당되는 변수의 원시값이 복사되어 전달된다.
```js
var score = 80;
var copy = score;

console.log(score); // 80
console.log(copy);  // 80

score = 100;

console.log(score); // 100
console.log(copy);  // 80
```

# 2. 객체

- JS의 객체는 클래스 없이 객체를 생성할 수 있다.
- JS의 객체는 생성된 이후라도 동적으로 프로퍼티와 메서드를 추가할 수 있다.

## 2-1. 변경 가능한 값

- 객체(참초) 타입의 값은 변경 가능한 값이다.
  - 객체를 할당한 변수가 기억하는 메모리 주소를 통해 참조 값에 접근한다.
  - 재할당 없이 객체를 직접 변경할 수 있다.
  ```js
  var person = {
    name: 'Lee'
  };

  person.name = 'Kim';
  person.address = 'Seoul';

  console.log(person);  // {name: "Kim", address: "Seoul"}
  ```
- 얕은 복사(shallow copy)
  ```js
  const o = { x: { y: 1} };

  const c1 = { ...o };      // false
  console.log(c1 === o);    // true
  ```
- 깊은 복사(deep copy)
  ```js
  // lodash 모듈을 사용한 깊은 복사
  const _ = require('lodash');

  const c2 = _.cloneDeep(0);
  console.log(c2 === 0);    // false
  console.log(c2.x === o.x);// false
  ```
## 2-2. 참조에 의한 전달

참조타입은 여러개의 식별자가 하나의 객체를 공유할 수 있다.

- call by value 와 call by reference는 식별자가 기억하는 메모리 공간에 저장되어 있는 값을 복사해서 전달한다는 점은 같다.
  - 다만 그 안의 값이 원시 값인지 참조 값인지에 따라 차이가 있다.
