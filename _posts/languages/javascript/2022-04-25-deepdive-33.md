# 심볼 기본

심볼은 자바스크립트의 새로운 기본 유형입니다.

숫자, 문자열, 부울이 있는 것 처럼 심볼이라는 기본 유형이 ES6 부터 추가됐습니다.

다른 값과 중복되지 않는 유일무이한 값으로, 객체안의 프로퍼티의 키 이름 충돌을 피하기 위해 쓰입니다.

```js
const mySymbol1 = Symbol();
const mySymbol2 = Symbol();

mySymbol1 === mySymbol2;

const mySymbol3 = Symbol("mySymbol");
const mySymbol4 = Symbol("mySymbol");

mySymbol3 === mySymbol4;
```

# 타입 변환

문자열, 숫자 타입으로는 변환되지 않지만 불리언 타입으로는 암묵적 변환이 가능합니다.

```js
const mySymbol = Symbol();

// 연결 연산자 +

mySymbol + '';

// 산술 연산자 +

+mySymbol;

// 논리 연산자 !
!mySymbol;
!!mySymbol;
```

# 메서드

`Symbol.for('key')`

- 전역 심벌 레지스트리에 해당 키로 새로운 심벌 값을 생성하거나 해당 키의 심벌 값을 반환합니다.

`Symbol.keyFor('key')`

- 전역 심벌 레지스트리에 등록된 심벌 값의 키를 추출 합니다.

```js
// Symbol.for
const s1 = Symbol.for("mySymbol");

const s2 = Symbol.for("mySymbol");

s1 === s2;

// Symbol.keyFor
Symbol.keyFor(s1);

const s3 = Symbol("foo"); // 전역 심벌 레지스트리에 등록안됨

Symbol.keyFor(s3);
```

# 활용

- enum

  ```js
  const ChatStat = {
    str: 0,
    img: 1,
    doc: 2,
    cam: 3,
  }
  // 관심사의 분리를 잘하고 사이드 이펙트 관리를 잘한다면 필요 없겠으나...

  const ChatStat = {
    str: Symbol('str'),
    img: Symbol('img'),
    doc: Symbol('doc'),
    cam: Symbol('cam'),
  }

  // 변경, 중복의 가능성이 없어 보다 안전함
  ```

- 프로퍼티 은닉
  - `for...in`, `Object.keys`, `getOwnPropertyNames` 메서드로 찾을 수 없음

  ```js
  const obj = {
    [Symbol('mySymbol')] : 1
  };
  
  for(const key in obj){
    console.log(key);
  }

  console.log(Object.keys(obj));
  console.log(Object.getOwnPropertyNames(obj));

  // 어? 그럼 열거되지 않는건가요?
  Object.getOwnPropertyDescriptors(obj);
  // Enumerable: true
  {...obj}
  ```

- 표준 빌트인 객체 확장
  ```js
  Array.prototype.remove = function() {
    this.splice(0, this.length);
  }

  Array.prototype[Symbol.for("remove")] = function() {
    this.splice(0, this.length);
  }
  // Symbol.for 주의!!!
  ```

- 이터레이션 프로토콜
  - `Symbol.iterator`를 키로 갖는 메서드를 객체에 추가하면 객체를 이터러블하게 만들 수 있음

  ```js
  // 1 ~ 5 범위의 정수로 이루어진 이터러블
  const iterable = {
    // Symbol.iterator 메서드를 구현하여 이터러블 프로토콜을 준수
    [Symbol.iterator]() {
      let cur = 1;
      const max = 5;
      // Symbol.iterator 메서드는 next 메서드를 소유한 이터레이터를 반환
      return {
        next() {
          return { value: cur++, done: cur > max + 1 };
        }
      };
    }
  };

  for (const num of iterable) {
    console.log(num); // 1 2 3 4 5
  }
  ```
