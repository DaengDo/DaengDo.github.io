# 1. 변수의 생명 주기

메모리 공간이 확보된 시점부터 메모리 공간이 해제되어 가용 메모리 풀에 반환되는 시점까지가 변수의 생명 주기이다.

- 지역 변수
  - 지역 변수의 생명 주기는 함수의 생명 주기와 일치한다.
  ```js
  function foo() {
    // 1. 변수 x 생성
    var x = 'local'; // 2. 변수 x에 값 할당
    console.log(x);
    return x;
    // 3. 변수 x 소멸
  }

  foo();
  console.log(x);
  ```
  - 호이스팅은 스코프를 단위로 동작한다.
  - 다만 어디선가 메모리 공간을 참조하고 있으면 할당된 메모리 해제되지 않고 유지하게 된다. (24장 클로저 참고)
- 전역 변수
  - 전역 변수의 생명 주기는 전역 객체의 생명 주기와 일치한다.

<BR>

# 2. 전역 변수의 문제점

- 암묵적 결합
  - 변수의 유효 범위가 넓을수록 가독성이 안좋아지고, 의도치 않은 상태 변경이 일어날 위험이 있다.
- 긴 생명 주기
  - 생명 주기가 길기 때문에 메모리 리소스도 오랜 기간 소비하며, 특히나 `var` 키워드로 선언한 변수는 재할당이 일어날 위험이 있다.
- 스코프 체인 최상위에 위치
  - 전역 변수는 스코프 체인 상 가장 마지막에 검색되므로 전역 변수의 검색 속도가 가장 느리다.
- 네임스페이스 오염
  - 파일 내에 동일한 이름의 식별자가 존재할 위험이 있다.

<BR>

# 3. 전역 변수의 사용을 억제하는 방법

전역 변수를 사용할 특별한 이유를 찾지 못했다면 지역 변수를 사용해야 한다. 변수의 스코프를 최대한 좁혀야 한다.

- 즉시 실행 함수를 적극 활용한다.
- 전역 변수가 필요한 경우 전역 네임스페이스 객체를 생성하고 사용하고 싶은 변수를 프로퍼티로 추가해서 사용한다.
  ```js
  var MYAPP = {}; // 전역 네임스페이스 객체

  MYAPP.name = 'Lee'; // 전역 변수 대신 프로퍼티로 활용
  ```
- 모듈 패턴을 활용한다. 서로 관련이 있는 변수와 함수를 모아 즉시 실행 함수로 감싸 하나의 모듈로 만든다.
- ES6 모듈을 활용한다. ES6 모듈은 파일 자체에 스코프를 제공한다.
  ```html
  <script type="module" src="lib.mjs"></script>
  <script type="module" src="app.mjs"></script>
  /* 모듈의 확장자는 mjs를 권장한다. */
  ```
