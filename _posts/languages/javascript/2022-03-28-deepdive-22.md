# this 키워드

`this`는 객체의 메서드가 자신이 속한 객체의 프로퍼티를 참조하기위한 자바스크립트의 키워드이다.

객체 리터럴 방식으로 생성한 객체의 경우 메서드 내부에서 자신이 속한 객체를 가리키는 식별자를 참조할 수 있지만, 생성자 함수로 생성된 인스턴스는 자신이 속한 객체를 가리킬 식별자를 알 수 없다.

```js
// 리터럴 방식으로 생성
const daengDo = {
  name: "DaengDo",
  sayHi(){
    console.log(`Hi! I'm ${daengDo.name}, nice to meet ya`)
  }
}

// 생성자 함수로 생성하는 인스턴스
function Person(name){
  ???.name = name,
}
Person.prototype.sayHi = function(){
  console.log(`Hi! I'm ${???.name}, nice to meet ya`)
}
```
`this`가 없다면 생성될 인스턴스의 식별자를 참조할 방법이 없다..

이처럼 `this`는 자신이 속한 객체 또는 자신이 생성할 인스턴스를 가리키는 자기 참조 변수다.

<BR>

# this 바인딩

`this` 는 함수 호출 방식에 의해 동적으로 결정되는데, 이를 `this` 바인딩이라고 한다.

- 객체의 메서드 내부에서 `this`는 메서드를 호출한 객체를 가리킨다.
- 생성자 함수 내부의 `this`는 생성자 함수가 생성할 인스턴스를 가리킨다.
- 전역에서 `this`는 전역 객체를 가리킨다. <BR>(this 사용의 의도와 어긋나서 strict mode 에서는 undefined가 출력된다.)
  - 일반 함수로 호출된 모든 함수 내부의 `this`에는 전역 객체가 바인딩된다.

그렇다면 메서드 내에서 선언한 모든 함수의 `this`는 해당 객체와 바인딩 될까?

```js
var value = 1;

const obj = {
  value: 100,
  foo(){
    console.log(`foo의 this : ${this}`);   // { value: 100, foo: f }
    console.log(`foo의 this.value : ${this.value}`); // 100

    function bar() { // 메서드 내의 중첩 함수
      console.log(`bar의 this : ${this}`); // 호스트 환경에 따른 전역 객체
      console.log(`bar의 this.value : ${this.value}`); // 1
    }
    bar();
  }
};

obj.foo();
```

메서드 내에서 정의했어도 일반적인 함수로 호출됐다면 전역 객체가 바인딩된다. 메서드의 콜백 함수 또한 마찬가지이다.

이 문제를 해결하려면 `this`를 변수에 할당하거나 `Function.prototype.apply`, `Function.prototype.call` 또는 `Function.prototype.bind` 메서드를 사용하면 된다.

```js
var value = 1;

const obj = {
  value: 100,
  foo(){
    console.log(`foo의 this : ${this}`);   // { value: 100, foo: f }
    console.log(`foo의 this.value : ${this.value}`); // 100
    
    const that = this; // this 바인딩(obj)을 변수 that에 할당

    function bar() { // 메서드 내의 중첩 함수
      console.log(`bar의 this : ${that}`); // { value: 100, foo: f }
      console.log(`bar의 this.value : ${that.value}`); // 100
    }
    bar();
  }
};

obj.foo();
```

앞선 중첩 함수의 `this`가 전역 객체로 바인딩되는 문제가 해결되었다.

다만 이 문제는 화살표 함수를 쓰면 골치아프게 고려할 필요가 없다. (...)

애시당초 화살표 함수의 `this`는 상위 스코프의 `this`를 가리키기 때문이다.

# apply, call, bind 메서드

`apply`, `call`, `bind` 는 Function.prototype 의 메서드로 사용법은 다음과 같다.

```js
Function.prototype.apply(thisArg[, argsArray])

Function.prototype.call(thisArg[, arg1, [arg2, [arg3, ...]]])

Function.prototype.bind(thisArg)
```

- `apply`와 `call`은 this로 사용할 객체와 인수를 받아 함수를 호출한다.
- `bind`는 this 바인딩이 교체된 함수를 새롭게 생성해 반환한다.

`apply`와 `call`은 유사 배열 객체에 배열 메서드를 사용하는 경우 유용하게 쓰인다.

```js
function convertArgsToArry() {
  console.log(arguments);

  const arr = Array.prototype.slice.call(arguments);

  console.log(arr);

  return arr;
}

convertArgsToArry(1, 2, 3);  // [1, 2, 3]
```
- 유사 배열 객체 `arguments`에 배열 메서드를 사용한 예시

`bind`는 중첩함수와 콜백함수에서 `this` 바인딩이 불일치하는 문제를 해결하는데 유용한 메서드이다.

```js
const person = {
  name : "Lee",
  foo(callback) {
    setTimeout(callback.bind(this), 100);
  }
}

person.foo(function (){
  console.log(`Hi! my name is${this.name}.`); // Hi! my name is Lee.
})
```
- person 객체의 메서드인 foo의 콜백함수의 this 바인딩을 person의 this 바인딩으로 통일 시켜준 예시


|함수 호출 방식|this 바인딩|
|--------------|-----------|
|일반 함수 호출|전역 객체|
|메서드 호출|메서드를 호출한 객체|
|생성자 함수 호출|생성자 함수가 생성할 인스턴스 |
|apply, call, bind <BR>메서드 호출|해당 메서드의 첫 번째 인수로 전달된 객체|
