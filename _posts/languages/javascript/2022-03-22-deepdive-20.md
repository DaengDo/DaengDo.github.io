strict mode는 자바스크립트 문법을 더 엄격하게 지켜 오류를 발생시킬 가능성이 높거나, JS 최적화 작업에 문제를 일으킬 수 있는 코드에 대해 에러를 발생 시킨다.

strict mode 에서 에러를 발생시키는 것은 아래와 같다.

<BR>

# strict mode 에서 일어나는 에러

## 암묵적 전역

선언하지 않은 변수를 참조하면 ReferenceError가 발생한다.

```js
(function (){
  'use strict';
  x = 1
  console.log(x); // ReferenceError: x is not defined
})()
```

<br>

## delete 연산자 제한

변수, 함수, 매개변수를 delete 연산자로 삭제하려고하면 SyntaxError가 발생한다.

```js
(function(){
  'use strict';

  const x = 1;
  delete x; // SyntaxError: Delete of an unqulified identifier in strict mode.

  function foo(a){
    delete a; // SyntaxError
  }
  delete foo; // SyntaxError
}())
```

<BR>

## 매개변수 이름의 중복

중복된 매개변수 이름을 사용하면 SyntaxError가 발생한다.

```js
(function(){
  'use strict';

  // SyntaxError: Duplicate parameter name not allowed in this context
  function foo(x, x){
    return x = x;
  }
  console.log(foo(1, 2));
})
```

<BR>

## with 문의 사용

`with` 문을 사용하면 SyntaxError를 발생시킨다. 애초에 with 문은 성능과 가독성이 안좋아 사용하지 않는 것이 권장된다고 한다. (언젠가는 deprecated 되지 않을까?)

```js
(function(){
  'use strict';

  with (Math){
    x = cos(3 * PI) + sin (LN10)  
    y = tan(14 * E)
  }
}())

// 아래처럼 씁시당
x = Math.cos(3 * Math.PI) + Math.sin(Math.LN10) 
y = Math.tan(14 * Math.E)
```

<BR>

# strict mode가 변화시키는 것

## 일반 함수의 this 바인딩

생성자 함수가 아닌 일반 함수에서 this는 사용할 필요가 없기 때문에 `undefined`가 바인딩 된다. (에러가 발생하지는 않음)

## arguments 객체

매개변수에 전달된 인수를 재할당하거나 변경해도 `arguments` 객체에 반영되지 않는다.

```js
(function (a) {
  'use strict';

  a = 2;
  console.log(arguments); // {1, length: 1}
}(1))
```

<BR>

그 외에 전역과 함수에서 strict mode 쓰는 것을 비권장한다고 한다.
