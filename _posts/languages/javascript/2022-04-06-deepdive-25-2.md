# 클래스의 인스턴스 생성 과정

클래스로 생성하는 과정은 다음과 같다.

1. 인스턴스 생성과 this 바인딩
2. 인스턴스 초기화
3. 인스턴스 반환

```js
class Person {
  constructor(name) {
    // 1. 인스턴스가 생성되고 this에 바인딩 된다.
    console.log(this);
    console.log(Object.getPrototypeOf(this) === Person.prototype);

    // 2. this에 바인딩되어 있는 인스턴스를 초기화한다.
    this.name = name;

    // 3. 완성된 인스턴스가 바인딩된 this가 반환된다.
  }
}
```

# 프로퍼티

- 인스턴스 프로퍼티는 `consturctor` 내부에서 정의한다.
- 접근자 프로퍼티 또한 클래스에서 사용이 가능하다. (getter, setter)
  - getter는 무언가를 반환해야하며, setter는 단 하나의 매개변수만 선언할 수 있다.

```js
class Person {
  constructor(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
  }
  get fullName() {
    return `${this.firstName} ${this.lastName}`;
  }
  set fullName(name) {
    [this.firstName, this.lastName] = name.split(' ');
  }
}

const me = new Person('Ungmo', 'Lee');

// 데이터 프로퍼티를 통한 프로퍼티 값의 참조.
console.log(`${me.firstName} ${me.lastName}`); // Ungmo Lee

// 접근자 프로퍼티를 통한 프로퍼티 값의 저장
// 접근자 프로퍼티 fullName에 값을 저장하면 setter 함수가 호출된다.
me.fullName = 'Heegun Lee';
console.log(me); // {firstName: "Heegun", lastName: "Lee"}

// 접근자 프로퍼티를 통한 프로퍼티 값의 참조
// 접근자 프로퍼티 fullName에 접근하면 getter 함수가 호출된다.
console.log(me.fullName); // Heegun Lee
```

# 클래스 필드 정의 제안

클래스 필드는 클래스 기반 객체지향 언어에서 클래스가 생성할 인스턴스의 프로퍼티를 가리키는 용어이다.

아직 ECMA 사양에 등록되지는 않았지만 TC39 프로세스의 3단계에 있는 클래스 필드는 chrome 72 이상 또는 Node.js 12버전 이상부터는 클래스 필드를 지원한다.

```js
class Person{
  isHandsome = false;
}

const you = new Person();
console.log(you); // Person {isHandsome: false}
```

클래스 필드에 함수를 할당할 수 있지만 모든 클래스 필드는 인스턴스 프로퍼티이기 때문에 프로토타입 메서드가 아닌 인스턴스 메서드가 되어 버린다. 따라서 함수를 클래스 필드에 할당하는 것은 적절하지 않아보인다.

클래스 필드를 초기화할 필요성이 있다면 `constructor`에서 인스턴스 프로퍼티를 정의하고,<BR>클래스 필드를 초기화할 필요성이 없다면 `constructor`와 클래스 필드 정의 제안 모두 사용 가능하다.

<BR>

# private 필드 정의 제안

클래스 필드와 마찬가지로 TC39 3단계에 있는 private 필드 정의 제안은 chrome74 이상 Node.js 12버전 이상부터 사용이 가능하다.

자바스크립트에는 `private` 키워드가 없기 때문에 클래스 필드는 언제나 항상 public하게 외부에 노출된다.

새롭게 구현될(아마) 자바스크립트의 private은 `#` 을 붙여서 사용한다.

```js
class Person {
  // private 필드 정의 (+ 클래스 필드 정의)
  #name = '';

  consturctor(name) {
    // private 필드 참조
    this.#name = name;
  }
}
const me = new Person("Daeng")

// private 필드 #name은 클래스 외부에서 참조가 불가능하다.
console.log(me.#name)
```

# static 필드 정의 제안

static 키워드를 사용해 정적 필드 또한 정의가 가능하다.(chrome 72, Node.js12 이상)

```js
class MyMath {
  // static public 필드 정의
  static PI = 22 / 7;

  // static private 필드 정의
  static #num = 10;

  // static 메서드
  static increment() {
    return ++MyMath.#num;
  }
}

console.log(MyMath.PI); // 3.142857142857143
console.log(MyMath.increment()); // 11
```

# 상속에 의한 클래스 확장

자바스크립트에서 클래스 문법을 사용하는 이유다. 생성자 함수는 문법적으로 지원하지 않지만 클래스는 `extends` 키워드로 상속을 지원한다.

```js
// 수퍼(베이스/부모) 클래스
class Base {}

// 서브(파생/자식) 클래스
class Derived extends Base{}
```
- 서브 클래스와 수퍼 클래스를 자바스크립트에서도 사용할 수 있다. 아래 예시를 보자.

```js
class Animal {
  constructor(age, weight) {
    this.age = age;
    this.weight = weight;
  }

  eat() { return 'eat'; }

  move() { return 'move'; }
}

// 상속을 통해 Animal 클래스를 확장한 Bird 클래스
class Bird extends Animal {
  fly() { return 'fly'; }
}

const bird = new Bird(1, 5);

console.log(bird); // Bird {age: 1, weight: 5}
console.log(bird instanceof Bird); // true
console.log(bird instanceof Animal); // true

console.log(bird.eat());  // eat
console.log(bird.move()); // move
console.log(bird.fly());  // fly
```
- 수퍼 클래스 Animal 을 서브 클래스 Bird 가 상속받아서 사용하는 예제이다.

놀랍게도 생성자 함수를 상속받아 클래스를 확장할 수도 있다.

```js
function Base(a){
  this.a = a;
}

class Derived extends Base {}

const derived = new Derived(1);
console.log(derived); // Derived {a: 1}
```

정확하게는 `[[constructor]]` 내부 메서드를 갖는 함수 객체는 다 사용할 수 있다.<BR>
(아래 예시 참고)

```js
function Base1() {}

class Base2 {}

let condition = true;

// 조건에 따라 동적으로 상속 대상을 결정하는 서브클래스
class Derived extends (condition ? Base1 : Base2) {}

const derived = new Derived();
console.log(derived); // Derived {}

console.log(derived instanceof Base1); // true
console.log(derived instanceof Base2); // false
```
- 이를 통해 동적할당도 가능하다.


서브 클래스에서 `constructor` 생략하면 
```js
constructor(...args) {super(...args);}
```
라는데 무슨 소리인지 모르겠음 발표하시는 분 발표 듣고 질문해야지...

# super 키워드

- `super`를 호출하면 수퍼클래스의 constructor를 호출한다.
- `super`를 참조하면 수퍼클래스의 메서드를 호출할 수 있다.

```js
// 수퍼클래스
class Base {
  constructor(a, b) { // ④
    this.a = a;
    this.b = b;
  }
}

// 서브클래스
class Derived extends Base {
  constructor(a, b, c) { // ②
    super(a, b); // ③
    this.c = c;
  }
}

const derived = new Derived(1, 2, 3); // ①
console.log(derived); // Derived {a: 1, b: 2, c: 3}
```
(인스턴스를 만듦에 있어 수퍼클래스와 서브클래스를 혼합해 활용한다.)

### super 사용 주의사항

- 서브클래스에서 `constructor`를 생략하지 않는 경우 서브클래스의 `constructor`에서 반드시 `super`를 호출해야함
- 서브클래스의 `constructor`에서 `super`를 호출하기 전에는 `this`를 참조할 수 없음

```js
class Base {}

class Derived extends Base {
  constructor() {
    // ReferenceError: Must call super constructor in derived class before accessing 'this' or returning from derived constructor
    this.a = 1;
    super();
  }
}

const derived = new Derived(1);
```

- `super`는 반드시 서브클래스의 `constructor`에서만 호출해야 함
