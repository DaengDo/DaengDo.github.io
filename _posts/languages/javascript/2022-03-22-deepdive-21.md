자바스크립트의 객체는 제작 환경에 따라 `표준 빌트인 객체`, `호스트 객체` 그리고 `사용자 정의 객체`로 분류할 수 있다.

`표준 빌트인 객체`는 ECMAscript 사양에 정의된 객체를 말하며 자바스크립트 실행 환경과 관계 없이 언제나 사용할 수 있다. 전역 객체의 프로퍼티로 제공되므로 별도의 선언 없이 전역 변수처럼 언제나 참조할 수 있다.

`호스트 객체`는 ECMAscript 사양에는 정의되어 있지 않지만 자바스크립트 실행 환경(브라우저, node.js 같은)에 따라 추가 제공되는 객체를 말한다.<BR>브라우저 환경에서는 DOM, BOM, Canvas, WebStorage 등 과 같은 클라이언트 사이드 Web API를 호스트 객체로 제공하고<BR>node.js 환경에서는 node.js 고유의 API를 호스트 객체로 제공한다.

`사용자 정의 객체`는 표준 빌트인 객체와 호스트 객체처럼 기본 제공되는 객체가 아닌 사용자가 직접 정의한 객체를 말한다.

# 표준 빌트인 객체

40여 개의 표준 빌트인 객체 중 `Math`, `Reflect`, `JSON` 을 제외한 나머지는 모두 인스턴스를 생성할 수 있는 생성자 함수이다.

```js
const strObj = new String("wow");
console.log(strObj);  // String {"wow"}
console.log(typeof strObj); // Object

const func = new Function("x, y", "return x + y");
console.log(func(1, 2)); // 3

const date = new Date();
```

# 래퍼 객체

원시값이 객체처럼 프로퍼티나 메서드를 사용하기 위해 래퍼 객체가 필요하다.
```js
const str = 'hello';

console.log(str.length); // 5
```
원시값이 객체처럼 접근하면 생성되는 임시 객체를 래퍼 객체라 한다.

```js
const str = 'hi';

// 원시타입인 str 이 String 인스턴스로 변환됨
console.log(str.length);  // 2

// 래퍼 객체로 프로퍼티나 메서드를 호출한 후 다시 원시값이 됨
console.log(typeof str);  // string
```
- String.prototype의 메서드를 상속받아 사용하는 예시이다.
