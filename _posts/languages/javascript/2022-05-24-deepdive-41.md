# 호출 스케줄링
  - 함수를 명시적으로 호출하지 않고 일정 시간이 경과된 이후 호출하는 것

- 타이머 함수
  - 타이머를 생성할 수 있는 `setTimeout`, `setInterval`와<BR>타이머를 제거할 수 있는 `clearTimeout`, `clearInterval`을 타이머 함수라고 한다.
  - 해당 함수들이 브라우저환경과 노드환경에서 사용이 가능해 빌트인 함수로 착각할 수 있으나,<BR>각각 환경에 구현이 된 호스트 객체이다.

- `setTimeout` / `clearTimeout`
  ```js
  const timerId = setTimeout(callback[, delay, param1, param2, ...])

  clearTimeout(timerId);
  ```
  - `callback`: 타이머가 만료된 뒤 호출할 콜백 함수
  - `delay`: 타이머 만료 시간(ms 단위), 인수 전달을 생략한 경우 기본 값 0
  - `param`: 호출 스케줄링된 콜백함수에 전달할 인수

  - 예제 코드 ↓
    ```js
    var timerId;

    // 타이머 등록
    function delayAlert(){
      timerId = setTimeout(globalThis.alert, 2*1000, '2초 뒤 실행!');
    }

    // 타이머 제거
    function clearAlert(){
      globalThis.clearTimeout(timerId);
    }
    ```

- `setInterval` / `clearInterval`
  ```js
  const timerId = setInterval(callback[, delay, param1, param2, ...])

  clearInterval(timerId);
  ```
  - `delay` 시간 동안 반복 호출된다는 점을 제외하면 `setTimeout`과 동일하다
  - 두 타이머 함수는 타이머 id를 반환하는데 브라우저 환경인 경우 숫자이고 Node 환경인 경우 객체이다.

# 디바운스

디바운스는 이벤트가 연속해서 발생했을 때,<BR>'일정 시간'이 경과한 이후 이벤트 핸들러가 '한 번만' 호출하게 한다.

`Lodash` 라이브러리의 `debounce` 함수로 편리하게 사용할 수 있다.

서버와 통신하는 인터페이스의 구현에서 유용하게 사용할 수 있다.

```js
_.debounce(callback, [wait=0], [options={}])
```

- 예제 코드 ↓
  ```js
  import _ from 'lodash'

  const sayHi = () => {
    console.log('hello!');
  }
  
  _action = _debounce(sayHi, 2000);

  _action();
  _action();
  _action();
  // 한 번만 호출된다!
  ```
