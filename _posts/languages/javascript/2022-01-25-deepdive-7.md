# 1. 산술 연산자

피연산자를 대상으로 수학적 계산을 수행해 새로운 숫자 값 생성
- 이항 산술 연산자
  - 2개의 피연산자를 산술 연산해 숫자 값 생성
  - `+`, `-`, `*`, `/`, `%`
- 단항 산술 연산자
  - 1개의 피연산자를 산술 연산해 숫자 값 생성
  - `++`, `--`, `+`, `-`
  - 증감 연산자는 피연산자의 **값을 변경**하는 부수 효과가 있음
    - 전위 증감 연산자: 피연산자의 값을 증감시킨 후, 다른 연산 수행
    - 후위 증감 연산자: 다른 연산 수행 후, 피연산자의 값을 증감
- 문자열 연결 연산자
  - `+` 연산자는 피연산자 중 하나 이상이 문자열인 경우 문자열 연결 연산자로 동작함<BR>(암묵적 타입 변환, 타입 강제 변환)

<BR><BR>

# 2. 할당 연산자

- 우항에 있는 피연산자의 평가 결과를 좌항에 있는 변수에 할당
  - [연산자 결합 방향](http://www.tcpschool.com/javascript/js_operator_arithmetic){:target="_blank"}이 오른쪽에서 왼쪽
- 할당문은 표현식인 문
  ```js
  var x;
  // 할당된 값으로 평가된다.
  console.log(x = 10); // 10
  ```

<BR><BR>

# 3. 비교 연산자

좌항과 우항의 피연산자를 비교한 후 그 결과 값을 `불리언 값`으로 반환

|비교 연산자|의미        |설명             |
|:----------|:-----------|:----------------|
|`==`       | 동등 비교  | 값이 같음       |
|`===`      | 일치 비교  | 값과 타입이 같음|
|`!=`       | 부동등 비교| 값이 다름       |
|`!==`      | 불일치 비교| 값과 타입이 다름|

- `==` 의 반대 개념은 `!=`
- `===` 의 반대 개념은 `!==`
- 동등 비교(`==`) 연산자는 암묵적 타입 변환을 통해 타입을 일치시킨 후 비교
  ```js
  5 == 5;   // true
  5 == '5'; // true
  ```
- `NaN`은 자신과 일치하지 않는 값<BR>`NaN` 여부 파악은 `Number.isNaN()` 을 활용
  ```js
  NaN === NaN;       // false

  Number.isNaN(NaN); // true
  Number.isNaN(10);  // false
  ```
- 숫자 0의 사례 주의
  ```js
  0 === -0 // true
  0 == -0  // true
  ```
- 비교적 예측 가능한 정확한 비교 결과를 반환하고 싶은 경우`Object.is()` 활용
  ```js
  -0 === +0;          // true
  Object.is(-0, +0)   // false

  NaN === NaN;        // false
  Object.is(NaN, NaN) // true
  ```
- 대소 관계 비교 연산자

  | 대소 관계 비교 연산자|설명             |
  |:---------------------|:----------------|
  |`>`                   | 좌항이 우항보다 크다|
  |`<`                   | 우항이 좌항보다 크다|
  |`>=`                  | 좌항이 우항보다 크거나 같다|
  |`<=`                  | 우항이 좌항보다 크거나 같다|

<BR><BR>

# 4. 삼항 조건 연산자

- 조건식의 평가 결과에 따라 반환할 값을 결정
  - 조건식 `?` 조건식이 true일 때 반환할 값 `:` 조건식이 false일 때 반환할 값
- 값으로 평가할 수 있는 표현식인 문(값으로 활용 가능)

<BR><BR>

# 5. 논리 연산자

| 논리 연산자 | 의미      | 단축 표현 |
|:-----------:|:----------|:---------|
|`||`         |논리합(OR) |왼쪽 true면 왼쪽 반환<BR>왼쪽 false면 오른쪽 반환|
|`&&`         |논리곱(AND)|왼쪽 true면 오른쪽 반환<BR>왼쪽 false면 왼쪽 반환|
|`!`          |부정(NOT) |불리언 값 반환(암묵적 타입변환)|

<BR><BR>

# 6. 쉼표 연산자

쉼표(`,`) 연산자는 왼쪽부터 오른쪽으로 피연산자를 평가 후<BR>
마지막 피연산자의 평가가 끝나면 마지막 피연산자의 평가 결과 반환
```js
var x, y, z;

x = 1, y = 2, z = 3; // 3
```

<BR><BR>

# 7. 그룹 연산자

- 소괄호(`()`)로 감싼 피연산자인 표현식을 가장 먼저 평가
- 연산자 우선순위가 가장 높아, 연산자의 우선순위를 조절 가능

<BR><BR>

# 8. typeof 연산자

- 피연산자의 데이터 타입을 문자열로 반환
- `string`, `number`, `boolean`, `undefined`, `symbol`, `object`, `function`<BR>총 7개 중 하나의 값 반환 (데이터 타입과 정확히 일치하지 않음)

<BR><BR>

# 9. 지수 연산자

- ES7 도입된 연산자로 지수 표현을 함
  ```js
  // 2의 3승
  2 ** 3; // 8
  ```
- 결합 순서는 오른쪽에서 왼쪽
  ```js
  2 ** 3 ** 2; // 512 가 출력(64가 아니다!!)
  ```
- 음수를 거듭제곱의 밑으로 사용할 시 그룹 연산자로 묶어야 함
  ```js
  -5 ** 2;    // SyntaxError

  (-5) ** 2;  // 25
  ```
- 이항 연산자 중 우선순위가 가장 높음
  ```js
  2 * 5 ** 2; // 50 (100이 아님!!)
  ```

<BR><BR>
