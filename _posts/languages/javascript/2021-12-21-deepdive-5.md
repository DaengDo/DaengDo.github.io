# 1. 값 (value)
- 값은 식(표현식, expresstion)이 평가(evaluate)되어 생성된 결과

<BR>

# 2. 리터럴 (literal)
- 값을 생성하기 위해 미리 약속한 표기법
  - 정수 리터럴, 부동소수점 리터럴, 2진수 리터럴, 배열 리터럴 등등등

<BR>

# 3. 표현식 (expression)
- 표현식은 값으로 평가될 수 있는 문<BR>표현식이 평가되면 새로운 값을 생성하거나 기존 값을 참조함
- 표현식과 표현식이 평가된 값은 동치

<BR>

# 4. 문 (statement)
- 선언문, 할당문, 조건문, 반복문 등으로 구분

<BR>

# 5. 세미콜론과 세미콜론 자동 삽입 기능
- ASI (automatic semicolon insertion)
  - ESLint와 TC39(ECMAScript 기술위원회)가 세미콜론 사용을 권장

<BR>


### 표현식과 문을 알아야 하는 이유

- 표현식의 역할은 값을 생성하는 것
- 문의 역할은 표현식으로 생성한 값을 키워드와 함께 사용해 JS엔진에 명령을 내리기 위한 것

<BR>

### 표현식인 문 vs 표현식이 아닌 문 (구분법)
- 변수에 할당했을 때 정상작동한다면 표현식 아니라면 문
  - 표현식은 무조건 값으로 변환되기 때문
