# 1. 동기/비동기 처리

- 동기 처리
  - 작성한 코드가 순서대로 동작한다.<BR>현재 실행 중인 작업이 종료될 때까지 다음에 실행할 작업이 대기하는 방식이다.
  - 실행 순서가 보장되지만, 현재 작업중인 작업 외의 작업들이 중단된다.
- 비동기 처리
  - 현재 실행 중인 작업이 종료되지 않은 상태여도 다음 작업을 곧바로 실행하는 방식이다.<BR>(엄청 추상화한 문장이군ㅋㅋ...)
  - 작업의 중단이 일어나지 않지만, 작업의 순서가 보장되지 않는다.
  - 타이머 함수와 HTTP요청, 이벤트 핸들러는 비동기 처리 방식으로 동작한다.
