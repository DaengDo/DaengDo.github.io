# 1. 동기/비동기 처리

- 동기 처리
  - 작성한 코드가 순서대로 동작한다.<BR>현재 실행 중인 작업이 종료될 때까지 다음에 실행할 작업이 대기하는 방식이다.
  - 실행 순서가 보장되지만, 현재 작업중인 작업 외의 작업들이 중단된다.
- 비동기 처리
  - 현재 실행 중인 작업이 종료되지 않은 상태여도 다음 작업을 곧바로 실행하는 방식이다.
  - 작업의 중단이 일어나지 않지만, 작업의 순서가 보장되지 않는다.
  - 타이머 함수와 HTTP요청, 이벤트 핸들러는 비동기 처리 방식으로 동작한다.

<BR>

# 2. 비동기 처리 순서

싱글 스레드인 자바스크립트 엔진이 동시성을 지원하기 위해 이벤트 루프의 도움을 받는다.

[MDN 이벤트 루프](https://developer.mozilla.org/ko/docs/Web/JavaScript/EventLoop){:target="_blank"}

(위 링크를 읽은 뒤 아래 예시 코드의 처리 순서를 한번 생각해보자){: .small}

```js
function foo() {
  console.log('foo');
}

function bar() {
  console.log('bar');
}

setTImeout(foo, 0);
bar();
```

- 1) 전역 코드가 평가되어 전역 실행 컨텍스트가 생성되고 콜 스택에 푸시
- 2) setTImeout 함수가 호출, setTimeout 함수의 함수 실행 컨텍스트가 생성되고 콜 스택에 푸시
- 3) setTImeout 함수가 실행되면 콜백 함수가 호출 스케줄링하고 종료되어 콜 스택에서 팝됨. 타이머 설정과 타이머가 만료되면 콜백 함수를 테스크 큐에 푸시
- 4) 브라우저, 자바스크립트 엔진 동시 처리
  - 4-1) 브라우저는 타이머가 만료되면 콜백 함수 `foo`가 4ms 후에 태스크 큐에 푸시, (0ms로 지정해도 최소 지연 시간인 4ms가 적용됨)
  - 4-2) `bar` 함수가 호출되어 `bar` 함수 실행 컨텍스트가 생성되고 콜 스택에 푸시, 이후 `bar` 함수가 종료되어 콜 스택에서 팝됨
- 5) 전역 코드 실행이 종료되고 전역 실행 컨텍스트가 콜 스택에서 팝됨, 콜 스택이 텅 비게 됨
- 6) 이벤트 루프가 콜 스택이 비어있는 것을 감지하여 태스크 큐에 대기 중인 콜백 함수 `foo`를 콜 스택에 푸시함. `foo` 함수 실행 컨텍스트가 생성되고 현재 실행 중인 실행 컨텍스트가 됨. 이후 `foo`함수가 종료되어 콜 스택에서 팝 됨.

차근차근 흐름에 따라 읽으면 쉽다. 요지는 콜 스택에 실행 컨텍스트가 하나씩 푸시되고 비동기 코드는 호스트 환경의 태스크 큐에 저장되며 콜 스택이 비어있을 때 이벤트 루프가 감지하여 태스크 큐에서 대기중인 함수를 콜 스택으로 옮겨준다. (FIFO로 옮김)

그럼 그렇지 완전 싱글 스레드로만 동작한다면 비동기를 구현할 수 없다...<BR>
호스트 환경(브라우저)과 어우러져서 비동기를 구현하는 것 이었다.<BR>
햇갈리지 말자 자바스크립트 엔진은 싱글 스레드로, 브라우저는 멀티 스레드로 동작한다!!!
