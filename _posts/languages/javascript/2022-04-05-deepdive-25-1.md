# 클래스 vs 생성자 함수
클래스는 함수이고 기존 프로토타입 기반 패턴을 클래스 기반 패턴처럼 사용할 수 있도록 해주지만 생성자 함수와 완전히 동일하게 작동하지는 않는다.

이하 서술된 점은 둘의 차이점이다.

- 클래스는 `new` 연산자 없이 호출하면 에러가 발생한다.
- 클래스는 상속을 지원하는 `extends`와 `super` 키워드를 제공한다.
- 클래스는 호이스팅이 발생하지 않는 것처럼 동작한다.
- 클래스 내의 모든 코드는 `strict mode` 가 자동 적용된다.
- 클래스의 constructor, 프로토타입 메서드, 정적 메서드는 모두 프로퍼티 어트리뷰트 `[[Enumerable]]` 값이 fasle 다. (열겨되지 않음)

<BR>

# 클래스 문법

- 클래스 선언문, 표현식(일반적이진 않음)으로 사용할 수 있다.

```js
// 선언문
class Person {}

// 표현식
const Person = class {};
const Person = class MyClass{};
```

- 클래스는 함수이므로 일급 객체로 활용할 수 있다.
- 클래스 몸체에는 0개 이상의 메서드만 정의할 수 있다.
  - 클래스 몸체에서 정의할 수 있는 메서드는 `constructor`, 프로토타입 메서드, 정적 메서드 3가지 이다.

```js
class Person {
  // 생성자
  constructor(name) {
    this.name = name;
  }

  // 프로토타입 메서드
  sayHi() {
    console.log(`Hi! My name is ${this.name}`);
  }

  // 정적 메서드
  static sayHello(){
    console.log(`Hello!`);
  }
}
```

비교를 위한 생성자 함수 예시 ↓
```js
var Person = (function(){
  // 생성자 함수
  function Person(name) {
    this.name = name;
  }

  // 프로토타입 메서드
  Person.prototype.sayHi = function () {
    console.log(`Hi! My name is ${this.name}`);
  }

  // 정적 메서드

})()

```
