# 1. 변수란 무엇인가? 왜 필요한가?
- 하나의 값을 저장하기 위해 확보한 메모리 공간 자체 또는 그 메모리 공간을 식별하기 위해 붙인 이름

<BR>

# 2. 식별자
- 어떤 값을 구별해서 식별할 수 있는 고유한 이름을 뜻함
- 식별자는 값이 아닌 메모리 주소를 기억하고 있음

<BR>

# 3. 변수 선언
- JS의 변수 선언은 `var`, `let`, `const` 키워드를 사용

<BR>

# 4. 변수 선언의 실행 시점과 변수 호이스팅

```js
console.log(score);

var score;
```

- 해당 소스코드 첫번째 줄은 참조 에러(Reference Error)가 발생하지 않고 `undefined`가 출력된다.
  - 변수 선언은 런타임이 아니라 그 이전 단계인 소스코드 평가 과정에서 일어나기 때문
- 호이스팅
  - 모든 선언문(`var`, `let`, `const`, `function`, `function*`, `class`)은 소스코드 평가과정에서 먼저 실행된다.

<BR>

# 5. 값의 할당

```js
var score;  // 변수 선언
score = 80; // 값의 할당
```
혹은 하나의 문으로 단축 표현
```js
var score = 80;
```
- `var` 키워드로 선언한 변수는 선언과 동시에 `undefined`로 초기화됨<BR>그 이후 값이 할당되는 것

<BR>

# 6. 값의 재할당

재할당은 현재 변수에 저장된 값을 버리고 새로운 값을 저장하는 것

```js
var score = 80;
score = 90;
```

엄밀히 말하자면 `var` 는 선언과 동시에 `undefined`로 초기화되기 때문에 변수에 처음으로 값을 할당하는 것도 사실은 재할당임

<BR>

# 7. 식별자 네이밍 규칙

- 특수 문자를 제외한 문자, 숫자, 언더스코어(`_`), 달러(`$`) 기호를 포함할 수 있음
- 특수문자를 제외한 문자, 언더스코어(`_`), 달러 기호(`$`)로 시작해야 함
- 숫자로 시작하는 것은 허용하지 않음
- 예약어는 사용할 수 없음
  - `strict-mode`가 아닐때에는 식별자로 사용가능한 예약어가(`private`, `public`, `let` 등) 몇 있음 (왠만하면 사용하지말자...)

### 7-1. 일반적인 네이밍 컨벤션
- 변수, 함수 -> `camelCase`
- 생성자 함수, 클래스 -> `PascalCase`
- 상수 -> `SNAKE_CASE`
