자바스크립트에 존재하는 데이터 타입은 총 8가지


|구분     |데이터 타입 |메모                    |
|:--------|:-----------|:----------------------|
|원시 타입|`string`    |                       |
|         |`number`    |64 비트 부동 소수점    |
|         |`bigint`    |ES11 추가, number보다 큰 정수 표현 가능|
|         |`boolean`   |                       |
|         |`undefined` |                       |
|         |`null`      |                       |
|         |`symbol`    |ES6 추가, 유일무이한 값|
|객체 타입|            |객체, 함수, 배열 등    |

사용 목적과 의도에 맞는 타입을 구별해서 값을 생성하고 취급해야 함

<BR><BR>

# 1. 숫자 타입

자바스크립트의 숫자 타입은 내부적으로 배정밀도 64비트 부동 소수점 형식을 따름
- JS 자체 2, 8, 16 진수 지원
  - `0b` : 2진수 표현
  - `0o` : 8진수 표현
  - `0x` : 16진수 표현
- 3가지 특별한 값
  - `Infinity` : 양의 무한대
  - `-Infinity`: 음의 무한대
  - `NaN` : Not a Number, 산술 연산 불가

<BR><BR>

# 2. 문자열 타입

- 텍스트 데이터를 나타내는데 사용
- `UTF-16` 기준을 따름
- 따옴표('', "", ``)로 감싸서 표현
- JS에서 문자열은 불변성을 지님(재할당은 가능)

<BR><BR>

# 3. 템플릿 리터럴

ES6 부터 추가된 문자열 표기법, 백틱(``` `` ```)을 사용함

- `이스케이프 시퀀스`를 사용하지 않고도 **줄바꿈**과 **공백**을 표현할 수 있음
- 표현식(expression)을 삽입할 수 있음 (표현식이 햇갈린다면 변수에 할당하여 확인)
  ```js
  // 템플릿 리터럴 사용
  console.log(`1 + 2 = ${1 + 2}`); // 1 + 2 = 3

  // 일반 문자열(따옴표) 사용
  console.log("1 + 2 = ${1 + 2}"); // 1 + 2 = ${1 + 2}    
  ```

<BR><BR>

# 4. 불리언 타입

`true`와 `false`를 사용해 논리 연산을 하는데 사용

<BR><BR>

# 5. undefined 타입

- var 키워드로 선언한 변수의 초기화에 사용되는 값
  ```js
  var foo;
  console.log(foo); // undefined
  ```
  - `let`과 `const`는 선언이 먼저 호이스팅되는 점까지는 `var`와 같지만, `undefined`로 초기화 되지는 않음
- 변수 초기화에 사용되는 값이므로 개발자가 의도적으로 할당하는 것은 올바른 사용법이 아님

<BR><BR>

# 6. null 타입

`null`
- 의도적 부재를 표현
  - 변수에 값이 없다는 것을 의도적으로 명시
  - 함수가 유효한 값을 반환할 수 없는 경우 명시

<BR><BR>

# 7. 심볼 타입

`Symbol()` ES6 추가 문법, 변경 불가능한 원시 타입 객체

- 다른 값과 중복되지 않는 유일무이한 값을 표현

<BR><BR>

# 8. 객체 타입

|원시 타입|객체 타입|
|:--------|:--------|
|변경 불가능한 값<BR>(immutable value)|변경 가능한 값<BR>(mutable value)|
|변수 할당시 `실제 값` 저장|변수 할당시 `참조 값` 저장|
|pass by value|pass by reference|

# 10. 동적 타이핑

- JS는 동적 타입 언어
  - 컴파일 시점의 타입 채크가 없음
  - 변수의 타입은 할당된 값의 타입에 따라 다름(타입 추론)
  - 변수의 타입은 재할당에 의해 언제든지 동적으로 변할 수 있음

- JS에서 변수 사용시 주의할 사항
  - 변수는 꼭 필요한 경우에 한해 제한적으로 사용
  - 변수의 유효 범위를 최대한 좁혀 부작용을 억제
    - 전역 변수는 최대한 사용을 자제
  - 변수보다 상수를 사용해 값의 변경을 억제
  - 변수 이름은 변수의 목적이나 의미를 파악할 수 있도록 네이밍
