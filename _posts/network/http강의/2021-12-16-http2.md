# URI 
- Uniform Resource Identifier
  - Uniform : 리소스를 식별하는 통일된 방식
  - Resource : 자원, URI로 식별할 수 있는 모든 것
  - Identifier : 다른 항목과 구분하는데 필요한 정보
- URN: Uniform Resource Name
- URL: Uniform Resource Locator
  - 구조 : `scheme://[userinfo@]host[:port][/path][?query][#fragment]`<BR>예시 : `https://www.google.com:443/search?q=hello&hl=ko`
  - `scheme:`
    - 주로 프로토콜(자원 접근 방식의 규칙, 약속) 사용,<BR>http는 `80`포트, https `443`포트를 주로 사용한다.
  - `userinfo`
    - URL에 사용자정보를 포함해서 인증,<BR>거의 사용하지 않음
  - `host[:port]`
    - 호스트명,<BR>도메인명 또는 IP주소 직접 사용 가능
  - `:port`
    - 접속 포트,<BR>http는 80, https는 443이 기본 포트로 일반적으로 생략
  - `/path`
    - 리소스 경로<BR>계층구조로 표현함
  - `query`
    - key, value 형태로 웹 서버에 제공하는 파라미터,<BR>query parameter, query string 으로 불림
  - `fragment`
    - 서버에 전송하는 정보가 아님,<BR>html 내부 북마크 등에 사용함

# 웹 브라우저 요청 흐름
- 웹 브라우저 → (소켓 라이브러리) <BR>→ TCP/IP → LAN 드라이버, LAN 장비 <BR>→ 인터넷 → 서버
