# scanf() 함수의 기본적인 사용법

## printf() 복습

- printf는 print formatted 로 `printf("format_specify")`

## scanf()

- scanf("format_specify", `&`i) , &(ampersand)를 꼭 붙여야한다.
  - call by reference 개념으로, 요즈음의 프로그래밍 언어와 달리 C 는 함수가 한 번에 하나의 출력 밖에 못하기 때문에, scanf() 가 여러 개의 입력을 받고 출력에 넘겨주는 상황 모두에 대응하기 위해 변수의 메모리에 직접 접근해서 바꿔주는 방식을 취했다.
- MS에서 scanf()를 활용할때는 오류가 뜬다.(MS에서만 이럼)
  - 해결방안 두가지 (전처리기 정의 두가지 방법)
    - 소스코드에 `#define _CRT_SECURE_NO_WARNINGS` 삽입
    - project → properties → preprocessor → configurations (debug) → preprocessor definitions → `_CRT_SECURE_NO_WARNINGS` 삽입 <br> configurations (release)에서도 똑같이 설정해주자.
- gcc 컴파일러는 오류가 뜨지 않는다.(ms의 vs에서만 오류가 남)

## scanf()가 MS에서 오류가 뜨는 이유

- 컴파일러가 컴파일을 하다가 scanf() 함수를 만나고 호출해 &(ampersand)를 입력받을때 문제가 발생한다.
- &(ampersand)는 변수의 주소를 알고싶을때 적는것이다.
- scanf는 변수의 값의 주소를 직접적으로 접근하는 것인데 그 주소를 에 해당하는 메모리의 값을 직접적으로 바꾼다.
- MS가 VisualStudio용 컴파일러를 만들때 직접적인 주소에 대한 접근이 보안에 위험이 된다고 생각되어 이런 오류를 출력하게되었다. (ms는 scanf대신에 scanf_s 함수를 이용하길 권장함)
