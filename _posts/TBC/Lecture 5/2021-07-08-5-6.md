# 연산자 우선 순위와 표현식 트리

- 컴퓨터 내부적으로는 연산할때 표현식 트리를 따라간다.

- 연산자 우선 순위는 아래 표 순서(위에서 아래)대로이다.

  | 연산자         | 결합법칙        |
  | -------------- | --------------- |
  | ( )            | 왼쪽에서 오른쪽 |
  | + -(단항unary) | 오른쪽에서 왼쪽 |
  | \* /           | 왼쪽에서 오른쪽 |
  | + -            | 왼쪽에서 오른쪽 |
  | =              | 오른쪽에서 왼쪽 |

  - 연산자(operator)괄호와 primary expression 괄호는 다르다. `( )` <br>그렇지만 괄호의 우선순위가 매우 높은 공통점이 있어서 그냥 뭉뚱그려 연산자로 표기하는 책이 많은듯 하다.
    - operator 는 함수 호출, 형 변환 연산자로 이용할때 쓰는 것
      - function(void) 할때 나오는 괄호
    - primary expression 은 식에서 쓰는 괄호
      - ex) &nbsp; -(1 + 2) \* 3 할때 나오는 괄호
