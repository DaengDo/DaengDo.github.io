> c언어 공부 목적

- c언어를 제대로 공부하지 못한다면 후속 과목들을 공부할때 문제가 생긴다.
- c언어를 공부하면 컴퓨터의 작동원리를 체득한다.

> 컴퓨터 작동 원리를 알아야 하는 이유

- 컴퓨터의 작동원리를 알지 못하면 프로그래밍을 하다가 하드웨어적인 지식이 부족해 막히는 부분이 있다.  
  _현대 프로그래머의 입장에서 꼭 알아야하는 중요한 하드웨어 개념을 익혀보겠다._

## 컴퓨터의 기능과 구성요소

> 컴퓨터의 기능

- 사용자가 입력장치를 활용해 컴퓨터에 정보를 제공
- 컴퓨터가 사용자가 의도한 기능을 수행
- 출력 장치를 통해 사용자에게 결과를 입력

> 컴퓨터의 구성요소

메인보드, 중앙 처리장치(cpu), 주 기억장치(rem), 보조 기억장치, 그래픽스 카드, 입출력 장치, 디스플레이

1.  중앙 처리 장치 ( Central Processing Unit )
2.  주 기억장치 (Primary Storage - Computer Memory)

    - 전원을 끄면 데이터가 사라진다. (Random-Access Memory의 특징)

3.  그래픽스 카드

    - GPU가 존재한다. (그래픽스 용도의 cpu와 메모리 존재)

4.  보조 기억장치(Secondary Storage)

    - 컴퓨터를 꺼도 데이터가 유지되지만, 주 기억장치보다 훨씬 느리다.

## 컴퓨터를 켤 때 일어나는 일들

> 컴퓨터를 키는 과정

1.  전원
2.  전원 공급 장치
3.  ROM BIOS (read only memory, basic input output system)

> 부팅 절차

1.  전원 공급
2.  부트 프로그램 실행(rom bios)
3.  하드웨어 검사
4.  운영 체제 로드 - 보조 기억장치에서 OS를 복사하여 주 기억장치에 전달
5.  운영 체제 실행 - 주 기억장치에 있는 프로그램(os)를 cpu가 실행

## 운영체제가 하는 일

> 운영체제

- 공통적 개발 항목인 하드웨어의 사용방법에 대한 부분을 만들어 놓은 일종의 틀이라고 볼 수 있다.  
  (옛날 프로그래머들은 모든 프로그램을 만들때마다 하드웨어를 어떻게 사용해야하는지 일일히 개발했다고한다)
- 컴퓨터의 리소스를 제어한다.

## 현대적 컴퓨터의 기본구조

## 컴퓨터 메모리의 기본 구조

cpu는 연산, 메모리는 명령어와 데이터를 저장한다.

캐쉬 메모리 : cpu에 자주 보내는 정보를 저장해두고 쓰는 개념(매번 같은 정보를 보내면 비효율적이니까) // 추가 공부 필요

- 순차 접근 : 원하는 데이터를 찾기위해 모든 부분을 다 찾는 것
- 임의 접근 : 원하는 데이터를 찾기위해 특정된 위치를 가서 얻는 것

## CPU의 기본 구조

> 프로그램이 시작되는 과정

- 보조기억 장치에 저장된 프로그램을 주기억장치로 옮기고 CPU로 연산하여 처리한다.

> cpu의 언어

- cpu는 모든 정보를 2진수의 숫자(기계어, machine code)로 바꿔서 처리한다.
  - 인간의 관점에서는 사용이 불편한 언어이기 때문에 이를 보완하기 위해 나온 것이 어셈블리어다.
  - 고수준 언어는 인간 친화적인 언어로 어셈블리어보다도 더 이해하고 운용하기 쉽다.

> cpu 명령어 집합

- cpu가 수행하는 기본적인 명령들은 많지 않다. 숫자를 더하거나, 숫자를 갖고오거나, 숫자를 저장하는 것과같은 단순한 작업들을 조합해서 복잡한 기능을 수행한다.

> cpu의 구성 요소들

- 제어 장치, 산술논리장치(ALU), 여러가지 레지스터들로 구성된다.
