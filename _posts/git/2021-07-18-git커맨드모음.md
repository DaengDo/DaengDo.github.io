## 깃 시작하기

- `git init`
  - 깃이 해당 폴더 내부의 변화를 기록 시작
- `git config --global user.name "{내 이름}"`
  - 커밋 작성자의 {내 이름} 입력
- `git config --global user.email "{내 이메일}"`
  - 커밋 작성자의 {내 이메일} 입력

<br>
<br>

## 히스토리와 상태 채크

- `git status`
  - 해당 폴더 깃의 변화 추이 확인
- `git log`
  - 커밋 히스토리 출력, 이곳에서 커밋의 해시값을 볼 수 있음( 헤드 이동시킬때 앞 6자리 채크 )

<br>
<br>

## 깃 저장하기

- `git add -A`
  - 모든 변경사항 저장
- `git add {특정파일확장자포함이름}`
  - {특정파일확장자포함이름} 저장
- 'git add -f {파일}
  - ignore 목록에 있어도 강제로(forced) add
- `git commit -m "{커밋할 내용}"`
  - {커밋할 내용} 커밋

<br>
<br>

## 브랜치 (branch)

- `git branch`
  - 로컬의 브랜치 목록 출력
- `git branch -r`
  - 원격의 브랜치 목록 출력
- `git branch -a`
  - 로컬/원격의 브랜치 목록 출력
- `git branch {새로운브랜치}`
  - {새로운브랜치} 추가
- `git switch -c {새로운브랜치}`<br>`git checkout -b {새로운브랜치}`
  - {새로운브랜치} 추가 후 HEAD 도 같이 이동
- `git branch -m {변경전브랜치이름} {변경할브랜치이름}`
  - 로컬 서버의 브랜치 이름 변경 <u>_앞에 -m이 무슨 의미일까?_</u>
- `git push {원격저장소이름} {변경할브랜치이름}`<br>`git push {원격저장소이름} --delete {변경전브랜치이름}`
  - 원격 서버의 브랜치 이름 변경, 이 두줄을 합처서
    - `git push {원격저장소이름} :{변경전브랜치이름} {변경할브랜치이름}`
- `git rebase {아래로 들어갈 브랜치 이름}`
  - 해당 {아래로 들어갈 브랜치 이름} 아래로 현재 채크아웃된 브랜치를 붙이기
- `git merge {합칠 브랜치 이름}`
  - 현재 채크아웃된 브랜치와 {합칠 브랜치 이름} 브랜치의 변화를 모두 합처서 현재 채크아웃된 브랜치에서 새로 커밋

<br>
<br>

## 상대참조 (relative reference)

- `^`
  - 한 커밋 위로 움직이는 명령어 <br> ex) `git checkout main^` 을 입력시 main 브랜치의 한 단계 위의 커밋으로 헤드가 이동
- `~{num}`
  - 한번에 여러 커밋 위로 움직이는 명령어 <br> ex) `git branch -f main HEAD~2` 을 입력시 main 브랜치가 강제로 HEAD 두 단계 위의 커밋으로 main 브랜치가 이동

<br>
<br>

## 소스트리 이동하기

- `git switch {이동할브랜치}`<br>`git checkout {이동할브랜치}`
  - {이동할브랜치} 로 HEAD 브랜치 이동
- `git branch {이동할브랜치} {특정커밋위치}`
  - {이동할브랜치}를 {특정커밋위치}로 HEAD 이동

<br>
<br>

## 특정 커밋 갖고오기

- `git cherry-pick {갖고올커밋} {갖고올커밋2} ...`
  - 헤드 아래에 원하는 커밋인 {갖고올커밋} {갖고올커밋2} ...을 추가
- `git rebase -i {HEAD~{num}}` (interactive rebase)
  - 현재 채크아웃된 브랜치의 헤드를 {HEAD~{num}} 만큼 뒤에서 rebase <br/> 원하는 커밋을 제외(omit) 하거나 추가(pick) 하거나 순서를 바꿀 수 있음

<br>
<br>

## 작업 되돌리기

- `git reset {특정 커밋}`
  - 해당 {특정 커밋}으로 HEAD를 이동 후, 이후의 변화를 삭제
- `git revert {특정 커밋}`
  - 해당 {특정 커밋}의 변경을 되돌린 후, 그대로 커밋

<br>
<br>

## 깃 서버와 통신

- `git push`
  - 원격 서버에 깃 데이터 업로드
- `git push -u {원격서버이름} {브랜치이름}`
  - 현 브랜치에 커밋된 내용들을 {원격서버이름}의 {브랜치이름}에 업로드<br><u>-u가 뭘 의미하는지 모르겠음 안써도 되던데...?</u>
- `git pull`
  - 현재 HEAD가 있는 브랜치에 remote되어 있는 원격 저장소의 브랜치 갖고오기
- `git pull {원격저장소이름} {원격저장소브랜치이름}`
  - git pull 명령에 특정 브랜치를 pull 해오는 방법<br>현재 작업중인 브랜치와 merge 시키는 실수가 나올수도 있으므로 유의
- `git clone {server_URL}`
  - {server_URL}의 깃 정보를 복사하여 갖고오기
- `git remote`
  - 현재 등록된 원격 서버 출력
- `git remote add {원격저장소이름} {원격저장소URL}`
  - 원격저장소의 이름과 URL 등록
- `git fetch`
  - 원격 서버의 변경점 채크
- `git checkout -t {원격저장소이름} {원격저장소브랜치이름}`
  - 원격저장소의 브랜치를 로컬에도 생성한 뒤 이동
- `git checkout -b {생성할 브랜치 이름} {원격 저장소의 브랜치 이름}`
  - 원격저장소의 브랜치 이름을 변경하여 로컬 브랜치로 갖고옴
- ` `

## `git ignore` (깃 제외 파일 설정)

- {파일이름}.{파일확장자}
  - 특정 파일 제외<br> ex : test.txt
- \*.{파일 확장자}
  - 특정 확장자 파일 제외<br>ex : \*.txt<br>
- {폴더이름}/
  - {폴더이름} 제외<br>ex : test/
