# 1. 최초 설정

## 1-1. 사용자 정보

`git config --global user.name "{사용자 이름}"`

`git config --global user.email "{사용자 이메일}"`

- 커밋할 떄마다 이 정보를 사용, 한 번 커밋한 후에는 정보 변경이 불가능하다.
- 해당 시스템에서 해당 사용자가 사용할 때는 이 정보를 사용한다
- 다른 이름과 이메일 주소를 사용하고 싶으면 --global 옵션을 빼고 명령을 실행한다.

## 1-2. 편집기

`git config --global core.editor {실행파일의 경로}`
- 나는 vscode 로 설정했다.

## 1-3. 설정 확인

`git config --list`
- 설정한 모든 것을 보여준다.

`git config <key>`
- 특정 key에 대해(user.name, core.editor같은 값) 어떤 값을 사용했는지 확인할 수 있다.

<BR>

`git config --show-origin rerere.autoUpdate file:/home/johndoe/.gitconfig	false`
> git이 설정된 값을 읽을 때 여러 파일에서 동일한 키에 대해 다른 값을 설정하고 있을 수 있다. 키에 설정된 값이 어디에서 설정되었는지 확인하는 명령어라고 한다.<BR>(안써봐서 무슨 소리인지 감이 안옴 써봐야 감이 올듯)
